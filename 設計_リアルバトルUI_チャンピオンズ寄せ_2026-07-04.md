## 2. スクショ取材まとめ(18枚・確定した実機仕様)

### 2.1 ステータスバー(statbar)

**相手(右上)**:
- クリムゾン(ピンク寄り赤)の**平行四辺形**パネル。左に丸いミニスプライトアイコン。
- 名前=白字+性別♂(青)/♀(赤)、下段に**緑のHPバー+「100%」大きめ表示(%のみ・数値なし)**。
- バーの下の行: 左=⏱持ち時間(例 06:57)、右=**残りポケモンのボールマーク**(緑モンボアイコン×残数)。

**自分(左下)**:
- 青紫の平行四辺形パネル。構造は相手と同じだが**HPは数値「171/171」**(現在値が大きい白字・最大値が小さめ)。
- ボールマーク+⏱はバーの**上**の行に置かれる(相手と上下反転の関係)。

**HPバー色**: シリーズ標準(緑>50% / 黄20〜50% / 赤<20%)を踏襲。減少はアニメで滑らかに減る。

### 2.2 コマンド/技選択(右側縦配置)

- **コマンドフェーズ**: 右側に丸い大ボタン縦並び「たたかう(緑)」「ポケモン(紫)」+黄色▶カーソル。上に **COMMAND + 残り秒カウントダウン**(例 44)。
- **技選択フェーズ**: 右側に**縦のカプセル型ボタン**(最大4)。各カプセル =
  - 左端の丸に**タイプアイコン**
  - 技名(白字・内側の濃い帯)
  - 右端にPP **8/8**
  - 名前の下に**相性ヒント帯**: 「◎ ばつぐん」/「△ いまひとつ」(等倍・変化技は帯なし)
  - 選択中 = **黄緑の発光縁取り**+左に黄色▶
- 技リストの上に **X 様子を見る / Y 技の説明** のボタンヒント。
- メガシンカ可能時は R+DNAアイコンのトグルが出る(実装済みのメガUIを流用し配置だけ寄せる)。

### 2.3 UIの出現/消滅タイミング(★阿部さん指定の肝)

チャンピオンズは**フェーズで全UIを丸ごと出し入れ**する:

| フェーズ | statbar | コマンド/技UI | メッセージ |
|---|---|---|---|
| コマンド選択中 | **両方表示** | 表示(COMMANDカウントダウン) | なし |
| 技/演出/メッセージ中 | **全部消える** | 消える | 表示 |
| 演出後のダメージ反映 | 再表示(バーが減る) | なし | 場合により継続 |

- メッセージ中は**statbarすら消える**(11.44.53/11.43.52で確認)。画面は3Dステージ+ポケモン+テキストのみ。
- 現実装(常時表示+右上かぶり)との最大の差はここ。**state machine で「選択中=UI表示 / 進行中=UI全消し」を切り替える**。

### 2.4 メッセージ表示

- **メッセージボックスなし**。画面下部左寄りに**白文字+黒フチ(アウトライン)**のテキストのみ。
- 2行構成が基本: 「相手の ソウブレイズの / つるぎのまい！」(行1=所有者、行2=技名+！)。
- 実機はふりがな付き(漢字の上にルビ)→ うちはひらがな主体の文なので**ルビは不要**(実装しない)。
- 特性発動も同形式: 「ジュペッタの アンコール！」「ジュペッタの みちづれ！」。
- ランク変化: 「相手の ソウブレイズの 攻撃が ぐーんと上がった！」。

### 2.5 選出画面(11.47.40)

- 左列 = 自分の6匹: 青紫の角丸カプセル(名前+性別/持ち物アイコン+右にスプライト)。選出済みに**左の数字バッジ(1,2,3)**、カーソル行は黄緑ハイライト+黄▶。下に「◉3/3 選出完了」ボタン。
- 中央メッセージ「戦うポケモンを 3匹 選出してください」。
- 右列 = 相手の6匹: 赤(クリムゾン)パネル(スプライト+タイプアイコン+性別)。
- 左上=⏱00:27カウントダウン、上中央=「ランクバトル シングルバトル」。右下=「Y つよさの表示ON」。
- → 現在のチーム編成モーダルの**将来の置き換え候補**。今回スプリントでは**バトル中UIを優先**し、選出画面は配色・カプセル形状だけ寄せる(構造替えは次段)。

### 2.6 攻撃モーション

- 実機はカメラズーム+3D演出 → **ズームはやらない(阿部さん指定)**。
- やるのは**スプライトの前方ランジ**(先に動く側が相手方向へ少し踏み込んで戻る、被弾側は白点滅+小シェイク=既存)。CSS transform 0.3s程度。

## 3. 実装仕様(real_battle.html 表示層のみ)

### 3.1 配色トークン

```css
--ch-opp-panel:   #d63384→#e0447e系 クリムゾングラデ;  /* 相手statbar/相手選出 */
--ch-self-panel:  #5b4fc9→#7b6ff0系 青紫グラデ;        /* 自分statbar/コマンド */
--ch-hp-green:    #9be82a;  --ch-hp-yellow: #f5c518;  --ch-hp-red: #e53e3e;
--ch-select:      #d4f542;  /* 選択中の黄緑発光 */
--ch-cursor:      #ffd400;  /* 黄▶ */
```

パネルは `clip-path: polygon()` か `transform: skewX(-12deg)` で平行四辺形化。

### 3.2 statbar 改修

- 相手(右上): 名前帯+HPバー+**%表示のみ**(`Math.ceil(hp/max*100)`+「%」・0にならない限り最低1%)。⏱は持ち時間制がないので**非表示**(枠は残さない)。ボールマーク=ベンチ残数(気絶除く)。
- 自分(左下): 同構造で**数値 `current/max`**。
- **右上の文字かぶり修正**: 現在右上に同居している要素(天候/ターン表示等)を statbar と重ならない位置(上中央の細帯)へ移設し、statbar自体は固定幅+`overflow:hidden;text-overflow:ellipsis` で名前が長くてもはみ出さないようにする。

### 3.3 UI出し入れ state machine

`uiPhase = 'command' | 'moves' | 'pokemon' | 'busy'`:
- `busy`(メッセージ/演出/ダメージ反映中) → `#cmd`/`#moves`/statbar 全てに `.ch-hidden`(opacity 0+pointer-events none・0.2s fade)。
- `command`/`moves`/`pokemon` → statbar+該当メニューを表示。
- ダメージ反映はメッセージ後に statbar だけ先に戻してバーを減らす(実機どおり)。

### 3.4 コマンド/技ボタン

- 既存の `#cmd`(たたかう/ポケモン)を**右側縦配置の丸ボタン**に。たたかう=緑・ポケモン=紫・黄▶は hover/選択で表示。
- `#moves` をカプセル型縦リストに: タイプアイコン(既存TYPE_COLORS利用の丸)+技名+PP右端+相性帯。
  - 相性帯 = 相手の現タイプ(sideTypes)に対する `typeEffectiveness`: >1「◎ ばつぐん」/ <1かつ>0「△ いまひとつ」/ 0「✕ こうかなし」/ 変化技・等倍=帯なし。**i18nキー9言語で追加**(`ui-battle` 系: `eff_super`/`eff_notvery`/`eff_no`)。
- COMMANDカウントダウンは**視覚だけ**(時間切れ強制行動はしない・シングル/ローカルなので圧を掛けない)→ 見送り可。まずは「COMMAND」ラベルのみ。

### 3.5 メッセージ

- メッセージボックス廃止 → 画面下部に `text-shadow` 多重黒フチの白文字(フォント太め)。
- **送り速度**: 既定を現「ふつう」より遅く(1行あたり表示保持 +400ms)。`#msg-speed` の選択肢を「ゆっくり/ふつう/はやい」に再調整し**既定=ふつう(新しい遅め値)**。
- 技コール行は「{所有者}の {技名}！」の2行組にできる場合は改行(既存ログ文字列は変えず、**表示時に整形**=translateLogLine後の表示層で改行挿入)。ログ文字列自体・i18n正規表現は**不変**(演出非破壊の原則)。

### 3.6 攻撃モーション

- 行動側スプライトに `.lunge-self`(右上へ 12px)/`.lunge-opp`(左下へ 12px)を付与→0.3sで戻す。ズームなし。既存の被弾フラッシュ/シェイクは維持。

### 3.7 やらないこと(今回)

- カメラズーム/3D演出・G-Max・選出画面の構造置き換え(配色だけ)・COMMAND実時間制限。
