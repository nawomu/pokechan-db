## 背景・目的(阿部さん提案 2026-07-01)
現状、**Champions本番DB**(`pokechan_data.js`=手書きcurated)と、**全国版DB**(`reference/*.json`→`pokechan_data_all.js`=PokeAPI由来)が**別々に存在**。このセッションの特性監査で、両者が食い違う(ABILITY_DESC 192 ↔ abilities_master 311・英語混入・第9世代追従漏れ)ことが実証された。

### ★突合で判明した最重要ポイント
1. **結合キーはJA名でなく id/slug**。JA名は Champions=半角 / 全国版(PokeAPI)=全角 で揺れる(`10まんボルト`↔`１０まんボルト`, `DDラリアット`↔`ＤＤラリアット`, `ＡＲシステム`↔`ARシステム`)。名前結合は壊れる。
2. **Champions固有("独自層")は薄く特定可能**: コインビーム(技)/連結特性2件/Champions独自特性(ドラゴンスキン等)/独自説明文/effects・battle_data/季(M-A/M-B)/メガ可否。=**全国版に重ねるoverlayとして保全すべき対象**。
3. **全国版が上位集合**。Champions ⊂ 全国版(表記正規化後)＋薄い独自層。

## 統一アーキテクチャ(提案)
```
[統一マスター(id/slugキー)]  種別ごと1本: pokemon / moves / abilities / items
  = 全国版(PokeAPI base: 名前9言語/種族値/タイプ/効果en 等)
  + Champions独自overlay(id/slugキー):
      - 独自エンティティ(コインビーム, 独自特性…)
      - 独自JA説明(マザー流) / effects・battle_data(sim用) / タグ / 体重 …
      - 属性: in_champions(季M-A/M-B), is_mega, is_original, legend, gen …
↓ ビルダー1本
[Champions view]  in_champions でフィルタ → 今と同じ形(POKEMON_LIST/WAZA_MAP/ABILITY_DESC/battle_data)を生成 → sim/本番はそのまま動く
[全国版 view]     マスター全体 → pokechan_data_all.js
```
- 肝: **Championsのsim/ページが読む"形"は変えない**(生成物として同じschemaを出す)。中身の出所だけ統一マスターに移す。=consumer(sim/HTML)を壊さない。
- 既に `_build_pokechan_data_all.js` は `pokechan_data.js`(curated)を土台にoverlayしている=**逆向き(統一マスター→Champions生成)に組み替える**のが本質。

## ★確定した構築フロー(阿部さん 2026-07-01)= 「全部版で作り直し→Champions抽出→差替」
**本番に合わせない。全部版をSSOTとして裏で完全に作り直し、そこからChampionsを導出して本番と差し替える。**
理由: composeエンジンが新しい=古い本番の焼き込み説明・独自move_no採番は陳腐化。合わせにいくと歪む。作り直しが綺麗。
```
1. [裏] 全部版(SSOT)を最新compose＋1本のeffects源で完全再生成
     - 説明文=最新composeで全919技を再生成(古い焼き込みは捨てる)
     - move_no=正規id/slugに統一(独自採番の7技も正規化)
     - 独自層(コインビーム/連結特性/季/メガ)はoverlayで保全
2. [裏] 全部版を全ページ＋simに通してテスト(下記テスト分担)
     - ポケモンDB全部版 / 技リスト全部版 / バトルシミュレーター
     - ★技はlistだけでなくsimで動かして初めて破綻が出る→sim全数スイープ必須
3. [裏] 全部版DBから「Championsの番号(id)だけ抜き出して」Champions版を生成
     - Champions = 全部版のビュー(in_champions/季でフィルタ＋旧schema互換で出力)
4. sim回帰テスト(既存505 + 全部版スイープ)でChampions挙動が破綻しないことを保証
```

注: 全部版のソース(reference/*.json + 現Champions curated overlay)には既にChampionsの手作り(effects/ABILITY_DESC/季/メガ/コインビーム)が入っている=**作り直しても curation は失われない**(突合で内容整合を確認済)。捨てるのは「陳腐化した焼き込み説明」と「独自move_no採番」だけ。

## ★ビルダー骨格(1本化)
```
tools/build_master.js        # SSOT生成
  master = PokeAPI base(reference/*_master.json)
         + overlay(effects=moves_battle_data_fix / desc source=ABILITY_DESC+desc_ja / flags / tags / champions属性 / 独自エンティティ)
  moves: description_ja = compose(effects)         ← 再生成(古い焼き込み破棄)
         move_no        = canonical id(slugで正規化・独自採番7技を実番号へ / コインビームは予約id)
  → reference/master_{pokemon,moves,abilities,items}.json

tools/build_champions_view.js   # master → 旧schema互換で出力(consumer無改変)
  filter master where champions.in==true
  emit → pokechan_data.js 形(POKEMON_LIST/WAZA_MAP{description,battle_data,...}/ABILITY_DESC)
  ★sim/本番ページはそのまま動く(形が同じ)

tools/build_national_view.js    # master 全体 → pokechan_data_all.js(現_build_pokechan_data_all.jsを置換)
```
**回帰ゲート**: build_champions_view の出力で `node tools/_sim_test.js`(505) + 全数simスイープが**pass維持**を確認してから本番差替。
