## §2 方針(2フェーズ)

### Phase 1: エディタで変化技を見える・調整できるように(本番の変化技挙動は不変)

1. **エディタのリストに変化技を出す**: ALL_MOVESのフィルタから'変化'を解禁(タブ/フィルタに「変化」も)。
2. **move側cueディスパッチ(_dispatchCueProd)に新glyphアクションを追加**(fx_primitives.js・省略時=既存cue不変):
   - `glyph/rank` {at, up:bool, stage:1-3} → rankFx(side, up, stage)
   - `glyph/sparkle` {at} → sparkleFx(side)
   - `glyph/wall` {name:'リフレクター'等, show:bool, at:'self'} → showWallFx/hideWallFx
   - `glyph/weather` {kind:'rain'|'sun'|'sand'|'snow'|null} → setWeatherFx(scene側と同語彙)
   - **`glyph/terrain` {kind:'grass'|'electric'|'psychic'|'misty'|null} → setTerrainFx(新プリミティブ・阿部さん追加指示)**: フィールドの見た目は現状未実装なので新設。地面(盤面下半分)からの演出=grass:葉っぱ🍃が舞い上がる+地面うっすら緑/electric:黄色いスパーク⚡が地を走る/psychic:紫の煙(もや)が立ちのぼる/misty:白ピンクの霧。実装は setWeatherFx と同型(#field内オーバーレイ+色味・null=解除)。設置(まきびし等)は今回対象外(見た目が小さい・後回し)。
   - `sound/se` を `p.name` 対応に拡張(scene側と同形・`p.name`無し=従来のSE.hitClass(cls)=後方互換)
   - **atkトラックは変化技テンプレに入れない**(誤生成バグの裏返し対策)。`at`既定は'opp'なので**自己バフ系cueは必ず{at:'self'}を明示**(罠)。
3. **エディタの自動生成=effectsからの分解**(class:テンプレ1枚でなく**技のeffectsを読んでcue化**):
   - 能力ランク変化 → glyph/rank(+SE rankUp/rankDown)×効果分(at=target側・stage=|stages|)
   - 回復 → glyph/sparkle + text/popnum('+N'緑)
   - 壁設置 → glyph/wall(show)
   - 天候変化 → glyph/weather(value→kind変換)
   - 状態付与 → def/flash(hit) + text/popnum('まひ！'等)
   - その他ロングテールkind → 汎用(sound/seのみ等)の最小cue+「このkindは自動分解未対応」の注記
   - ※класс:statusの単一テンプレは作らない(中身が多様すぎる=調査の結論)。`_cueClassKey`は変化技で''のまま。
4. **インスペクタ**: 新アクションのパラメータ欄(rank=up/stage・wall=name/show・weather=kind)。
5. **エディタプレビューの見た目=本番と同じ関数**(rankFx等を同じ引数で呼ぶ)なので視覚パリティは自動で取れる。

### Phase 2(阿部さんがPhase 1を触ってから判断): 本番も変化技をcueシートで上書き可能に
- 変化技の技使用行(`{name} の {技}！`)でresolveCueSheet(move:のみ・done:true)を引き、シートがあれば再生+**後続ログ行の正規表現fxを抑制**(二重演出防止)。シート無し=従来の正規表現経路(完全不変)。
- 対象はmove:個別上書きのみ(classデフォルト無し)=調整した技だけ変わる安全設計。
- **★text/popnumのJA固定文字のi18n(2026-07-16 阿部さん指摘)**: 自動分解が作る popnum の text(「みがわり！」「まひ！」「＋」等)はJA直書き。Phase1=エディタプレビュー専用なので無害だが、**Phase2で本番採用する時は必ずi18n対応**(案: textを生テキストでなくキー参照 {textKey:'substitute'} にして表示時にI18N.tで解決 or バトルログと同じtranslateLogLineパターン化。数字/記号だけ(＋・−42)は言語非依存なのでそのまま可)。i18n-first-designの原則どおり「作ってから探す」をしない。
