大原則

そのまま書く(最優先)
説明文は 構造データ(effects)の中身を、そのまま・言葉通りに 日本語にする。アレンジ・ひねり・まとめ・省略・やさしく崩す・AIの変な加工は 全部しない。子どもに分からなくても、まず事実をそのまま伝える。「やさしい子ども口調」より忠実が優先(softening が事実をぼかすなら忠実を採る)。「直訳」=データの逐語訳であって、子ども向けの作文・要約ではない。

言葉のルール(✗ → ✓)

ルール✗ しない✓ する
日本語がベース・正確で標準的な日本語
日本の会社の製品。公式準拠の用語・適切な漢字で。英語は英語のままでよい。
 — —
公式データ(全ひらがな・幼稚)に寄せない
一般的な日本語表現にする。
 ✗ きゅうしょにあたりやすい ✓ 急所ランクがひとつ上がる
事実をそのまま(解釈・softening禁止)
データの意味を勝手に言い換えない。
 ✗ 急所に当たりやすい ✓ 急所ランクがひとつ上がる
助数詞は和語数詞
ランクの増減は「ひとつ・ふたつ・みっつ…」。「1個」にしない。
 ✗ 急所ランクが1個上がる ✓ 急所ランクがひとつ上がる
数値はそのまま
確率などの数字は崩さない。
 ✗ 10回に1回くらい ✓ 10%の確率で
ヤックンとそっくり同一は避ける
必須でないが望ましい。忠実に訳した結果のbare一致(自然な収束)は可。なるべく被らせない。

機械語を出さない
true / 0.125 / 英語 / キー名 をそのまま出力に出さない(部品→日本語に)。
 ✗ until_user_leaves ✓ 自分が場を離れるまで
