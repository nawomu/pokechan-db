## 🧠 わざ分類 / バトルシミュレータ設計の思考パターン

### 1. 「両方アプローチ併用」が原則

**テキストパース + Phase ベース phrases** の両方を組み合わせる。  
どちらか片方に頼ると必ず漏れが出る:

- **テキストパース**: description / description_legacy を正規表現で解析
  - 例: `自身の(.+?)\s*([+\-])\s*(\d)` で「自身のXX+N」を全抽出
  - 複数要素ある技 (からをやぶる = 5要素) も全部拾える
- **Phase phrases**: 既存のキーワード一致 (能力変動以外の効果)
- **両方の結果をマージ** して、重複は1つに

### 2. 3段階判定でフォールバック

```
L1: テキストパースで自動抽出 (正規表現)
  ↓ 失敗
L2: 既存 phrases マッチ (Phase ベース)
  ↓ 失敗
L3: 両方で不明 → 外部情報源 (ネット検索) で確認 → 「不明」と明示
```

**「答えがわからない」と明示することが重要**。誤分類より「不明」マーク。  
迷ったらネット検索 (ポケモン徹底攻略 yakkun.com / 育成考察Wiki) を使う。

### 3. AND 条件は全タグ付与 (分解)

複数の効果を持つ技は、**各要素を独立タグとして全部付与**:

- からをやぶる = `self_def_down_1` + `self_spd_down_1` + `self_atk_up_2` + `self_spa_up_2` + `self_spe_up_2`
- ビルドアップ = `self_atk_up_1` + `self_def_up_1`
- くすぐる = `opp_atk_down_1` + `opp_def_down_1`
- ハバネロエキス = `opp_atk_up_2` + `opp_def_down_2` (敵の能力 +/- 同時)

**「&」(AND) を構造として持たせる**。並列タグで表現。

### 4. オンリーワン技は「その他」吸収

1件だけのフィルタになる固有効果は `other_misc` に集約:
- 例: おちゃかい (場の全ポケがきのみ即時発動) → `other_misc`
- フィルタの観点で「1件しか持たないタグは無意味」 → ある程度集まったらタグに昇格

### 5. 対象軸 (target_*) は別軸

能力変動・効果の対象を明示する独立軸:
- `target_self` (自分)
- `target_opp` (敵)
- **`target_ally`** (味方、ダブル用) ← 「自分以外の味方」軸を忘れない
- `target_all` (全体)
- `target_field` (場)

### 6. 「ダブル○○」は "only" ではなく "サポート用"

- ✅ 「ダブルサポート用 (シングル可)」
- ❌ 「ダブルバトル専用」

```
[戦闘開始時]
Init-A: 特性発動 (いかく、てんき特性、すなおこし等)
        ※ 速度順に発動 (Init-B の速度チェック含む)

[ターン終了処理 — 細分化]
10a: 状態異常スリップダメ (やけど/どく/もうどく/しおづけ)

10d: 場の継続回復 (グラスフィールド/アクアリング/やどりぎ)
10e: 時間差カウンタダウン (ねがいごと/みらいよち/ほろびのうた/ロックオン)
10f: ターン数カウンタダウン (天候/フィールド/壁/ルーム/技封じ)

10h: ロック継続 (あばれる/げきりん/さわぐ)
```

## 📐 ステータス計算ルール (能力ポイント方式)

**重要**: 実数値の計算は [HANDOFF_PARTY_CHECKER_STAT_POPUP.md](HANDOFF_PARTY_CHECKER_STAT_POPUP.md) を参照。
party_checker と完全に同じ式をシミュレータでも使うこと。

### 要点 (詳細は別資料)
- IV は 31 固定 (Champions 仕様)
- pts は EV/4 ではなく **直接 0–32** で扱う
- 性格補正は **(inner + pts) 全体に乗ずる** → `floor((inner + pts) × nature)`
- 1pt の効果: 中性 = +1 / 性格 ↑ = +1 or +2 / 性格 ↓ = +0 or +1
- `computeActualStat()` を party_checker から共通化して流用可能

## 🛠️ フェーズC-3 / C-4 実装計画 (2026-05-15 追記)

### 設計の前提

- **データソース**: `pokechan_data.js` の `WAZA_MAP` は既に各技に `tags` / `battle_data.effects` / `flags` を保有 (490技)
