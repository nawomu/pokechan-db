## §1 調査確定事実(全て実ファイルで裏取り済み)

### 不満Aの原因 = 三重構造

1. **CSSハードコード上限**: 炎バースト `.burst` は `width:120px;height:120px` + `@keyframes rbBurst{0%{scale(.15)}100%{scale(1.8)}}` = **最大直径216px固定**。同一定義が3ファイルに重複:
   - `real_battle.html:446-448`
   - `online_battle.html:552-554`
   - `fx_editor.html:195-197`(ツクールのプレビューも同じ上限)
2. **`params.scale` が完全デッド**: エディタの scale 入力欄(`fx_editor.html:1141-1142` 定義・`1185` bind、min0.1/step0.1/maxなし)は `cue.params.scale` に書くだけで、**リポジトリ全体に読み取り箇所ゼロ**。フレアドライブの本番cueには既に `"scale": 100` が入っている(SSOT `reference/battle_fx_cues.json:101` / ビルド産物 `battle_fx_cues.js:108`)が握りつぶされている。ディスパッチ `_dispatchCueProd` のburst分岐(`fx_primitives.js:855-859`)は `burstFx(atSide, color, shape, p.intensity||'normal', cue.dur)` で **p.scale を渡していない**。
3. **dur は時間にしか効かない**: `burstFx`(`fx_primitives.js:40-57`)の `scale = durMs/650`(44行)は animationDuration とタイマーにのみ使用。パーティクルは `4+Math.random()*4` px 固定(68行)、リング拡大比 `scale(.2)→scale(4)` 固定(95-96行)。**空間(px/倍率)に効く変数が一つも無い**。

### ついでに発見した配線漏れバグ(第1弾で同時修正)

- **offset が本番で効いていない**: `_dispatchCueProd`(`fx_primitives.js:856-858`)は `--fx-ox`/`--fx-oy` を `#f-<side>` に setProperty している(本番でも実行される)が、**それを消費するCSSが `fx_editor.html:195` にしか無い**。`real_battle.html:446` / `online_battle.html:552` の `.burst` は `left:50%;top:46%` 固定。→ ツクールで合わせた出現位置(フレアドライブは offset {x:-4, y:21})が**本番では反映されずズレる**。

### 不満Bの原因 = スクラブ機能の不在(モーション実装は正常)

- 突進モーション `_cueChargeMotionProd`(`fx_primitives.js:772-817`)は WAAPI 4キーフレームの**正しい連続補間アニメ**。本番と同一関数。紙芝居なのはモーションではなくエディタUI。
- エディタの再生 `scheduleRound`(`fx_editor.html:1413-1444`)は全cueを setTimeout 一括予約する**実時間再生のみ**。「時刻tの状態を描く」関数(renderAtTime/seek/scrub相当)は**リポジトリ全体に存在しない**(grep 0件)。
- ルーラードラッグ(`fx_editor.html:1511-1552`): 移動量≥4pxのドラッグは `setPlayheadPos()`(1531行)**のみ**=赤線が動くだけで盤面は無反応。cueバーのドラッグ/リサイズ(1263-1330行)も同様に盤面を更新しない。
- → 動きを見る唯一の手段が「▶で頭から実時間再生」。**時間軸を手でめくれない**のが紙芝居感の真因。

### 参考(現状の構造・変更時に知っておく)

- エディタと本番は `_dispatchCueProd`/`playCueSheet`(`fx_primitives.js:843-900`)を**完全共有**(エディタ側 `dispatchCue` は薄いラッパー `fx_editor.html:1341-1349`)。fx_primitives.js を触る=本番も変わる。
- プレビュー舞台は real_battle.html の DOM/CSS **複製**(iframeではない)。CSS変更は3ファイル同時に揃える必要あり。
- 描画方式: canvas不使用。DOM+CSS keyframe+WAAPI(`el.animate()`)。rAFは fieldShake のみ(trauma減衰モデル、`fx_primitives.js:421-456`)。
- popnum文字サイズ `p.size`(既定16、`fx_primitives.js:876`)と burst の `intensity` はインスペクタに編集欄が無い(今回のスコープ外・第2弾後の小物候補)。

## §2 第1弾: scale配線 + offset本番配線(小規模・先にやる)

### 2-1. scale を実際に効かせる

**意味の定義(重要)**: `params.scale` = **見た目倍率(乗数)。1.0 = 現状サイズ**。ディスパッチ側で `clamp(0.1〜6)` する(scale:100 のような残存値・入力事故で画面が壊れるのを防ぐ)。

1. **CSS(3ファイル同一変更)**: `.burst` を CSS変数で可変に。
   ```css
   .burst{ ... width:calc(120px * var(--fx-burst-scale,1)); height:calc(120px * var(--fx-burst-scale,1));
           margin:calc(-60px * var(--fx-burst-scale,1)) 0 0 calc(-60px * var(--fx-burst-scale,1)); ... }
   ```
   keyframe `rbBurst` の scale(.15→1.8) は倍率が width/height に掛かるのでそのままで良い。対象: `real_battle.html:446-448` / `online_battle.html:552-554` / `fx_editor.html:195-197`。
2. **JS配線**: `_dispatchCueProd` burst分岐(`fx_primitives.js:855-859`)で `p.scale` を読み、clamp して `burstFx` へ渡す。`burstFx(side, color, shape, intensity, durMs, sizeScale)` と第6引数を追加(**省略時=1=旧挙動と完全一致**。共有ファイルの機能追加は省略可能引数+省略時旧値一致、2026-07-11教訓)。`burstFx` 内で burst要素に `el.style.setProperty('--fx-burst-scale', sizeScale)`。
3. **サブ要素も倍率に追随**: `spawnBurstParticles`(粒サイズ 4-8px→×sizeScale、拡散距離も×sizeScale)/`spawnBurstRing`(リング要素の基準サイズ×sizeScale)/`spawnBurstGlyph`(グリフ要素に同じCSS変数 or fontSize×sizeScale)。glyph系CSSクラスのサイズ定義が3ファイルCSS側にあれば同様に var() 化。**時間用の既存 `scale`(=durMs/650)と混同しない**こと(変数名は sizeScale で分離)。
4. **SSOTの残存値を掃除**: `reference/battle_fx_cues.json:101` の `"scale": 100` は意図不明の暴走値(倍率として解釈すると100倍)。**2.0 に置き換え**て(まず1.5〜2倍で阿部さんがツクールで再調整する前提)、`node tools/_build_battle_fx_cues_js.js` で `battle_fx_cues.js` を再生成。**ビルド産物を直接編集しない**。
5. **エディタUI**: `fx_editor.html:1141` の scale 欄ラベル「(見た目倍率・書き出しのみ反映)」→「(見た目倍率 1=標準)」に修正。max=6 を input に付ける。プレビューでも即時反映されること(同じディスパッチを通るので配線すれば自動で効くはず)。

### 2-2. offset を本番でも効かせる

- `real_battle.html` / `online_battle.html` の `.burst` の left/top を fx_editor と同じ
  `left:calc(50% + var(--fx-ox,0px)); top:calc(46% + var(--fx-oy,0px));` に変更(2-1のCSS変更と同時に)。
- 注意: `--fx-ox` は `#f-<side>`(fighter要素)に set されるので、`.burst` がその子孫にある構造が3ページで同じことを確認してから適用。

### 2-3. 第1弾の検証ゲート(PDCA・満たすまで報告しない)

1. `node --check fx_primitives.js` 等の静的検査 → ローカルサーバ(`python3 -m http.server 8000`)+ **Playwright実機**。
2. **3ページ回帰**: real_battle.html / online_battle.html / fx_editor.html それぞれで
   - JSエラー0
   - フレアドライブ(cueシートあり経路): 炎が scale=2 で明確に大きく出る+offset位置が3ページで一致
   - はたく(cueシートあり・scale未指定): **従来と同一サイズ**(=省略時1.0の後方互換)
   - cueシートが無い技(フォールバック `attackFx` 経路): 従来どおり
3. スクリーンショットを撮って before/after を報告に添付。
4. `?v` bump 必須: fx_primitives.js(現 20260713c)/ battle_fx_cues.js(現 20260713a)→ 20260715a。**3ページすべての読み込みタグを揃えて bump**([[fx-editor-cue-integration]]の罠)。

## §3 第2弾: タイムラインスクラブ(中規模・第1弾の後)

**ゴール**: 赤いプレイヘッドをドラッグすると、その瞬間の絵(突進の途中位置・炎の膨らみかけ)が盤面に出る。AfterEffects/iMovie的な「時間軸を手でめくる」体験。

### 3-1. 段階A: スナップショット簡易版(まず出す・fx_editor.html内で完結)

- ルーラードラッグの `onMove`/`onUp`(`fx_editor.html:1519-1552`)にスクラブ描画を追加:
  1. 盤面リセット(既存の周回頭リセット処理を関数化して流用)
  2. `currentSheet.cues.filter(c => c.t <= targetMs)` を時系列順に同期 dispatch
  3. ドラッグ中は間引き(rAF1回に1度)
- 限界(承知の上): 進行中cue(t < targetMs < t+dur)の「途中状態」は出ない(未発火 or 最終状態)。それでも「どの時刻に何が出てるか」は見えるようになる。
- **fx_primitives.js には触れない**(本番無風)。

### 3-2. 段階B: 本物のシーク(段階Aの使い心地を阿部さんに確認してから)

- 原理: 主要プリミティブは既に WAAPI(`el.animate()`)なので `Animation.currentTime` でシーク可能。CSS class駆動のアニメも `el.getAnimations()` で Animation オブジェクトが取れる。
- エディタ専用のスクラブレンダラ(fx_editor.html側)方式を推奨:
  1. スクラブ時刻 t について、`c.t <= t < c.t+dur+余韻` の cue を dispatch
  2. dispatch直後に `document.getAnimations()`(dispatch前後の差分)で生成された Animation を捕まえ、`anim.currentTime = t - c.t; anim.pause();`
  3. プリミティブ内の `setTimeout` 自己破棄が発火する前に静止画として見せる(スクラブ中は連続再dispatchなので実用上問題になりにくいが、破棄タイマーが邪魔になる場合はエディタ側でグローバルフラグ `window.__FX_SCRUB__` を見て setTimeout をスキップする小改修を fx_primitives.js に入れる=省略時旧挙動)
  4. rAF駆動の fieldShake と `#field.style.filter` 直操作系はシーク不能 → スクラブ中はスキップ(揺れはプレビュー再生でのみ確認)
- **fx_primitives.js を改修する場合は本番3ページの回帰(Playwright+スクショ)必須**。改修は「スクラブフラグで分岐」の最小侵襲に留め、本番経路のコードパスは不変にする。
- cueバーのドラッグ/リサイズ中も同じスクラブレンダラを `cue.t`(または `cue.t+dur`)時刻で呼ぶと「バーを動かす=絵が追随」になり編集感が一気に上がる(段階Bのおまけ・工数小)。

### 3-3. 第2弾の検証ゲート

- fx_editor.html: JSエラー0/ドラッグで盤面が追随(スクショ数点)/▶通常再生・ループ・スロー・途中クリック再生が**従来どおり壊れていない**こと。
- fx_primitives.js に触れた場合: real_battle / online_battle でフレアドライブ+はたく+cueなし技+交代(recallFx)+メガシンカの回帰。
- 使い心地の最終判定は阿部さん(段階A→Bの進むかどうかも阿部さんの耳ならぬ**手**)。

## §4 既知の罠(再発注意・[[fx-editor-cue-integration]]より)

- **body-fixed clone の scale/zoom 焼き込み**(`fx_primitives.js:320-326, 786-803`): 突進クローンは `#f-*` の奥行きscale+`#field` zoom を実測比で焼き込んでいる。burst のサイズ変更でここに手を出す必要はないが、**glyph/burst を body直下に出す改修は絶対にしない**(2026-07-13の左上ワープ事故の轍)。
- **flash() は800ms自動解除**(`fx_primitives.js:458-464`): 800ms超の演出に flash() を使うな(rbRecall殺し事故)。
- **#field へのクリックバブリング**: エディタに新しいクリックUIを足すときは stopPropagation を確認。
- **ビルド産物直編集禁止**: cue の SSOT は `reference/battle_fx_cues.json` → `node tools/_build_battle_fx_cues_js.js` で `battle_fx_cues.js` 生成。
- **エージェント編集中ファイルは git add しない**([[agent-file-commit-lockout]])。
- **`?v` bump 忘れ**: fx_primitives.js / battle_fx_cues.js を変えたら3ページの `?v` を必ず揃えて上げる。
