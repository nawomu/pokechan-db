# ヤックン耳 — 判断ログと境界線

最終更新: 2026-06-05

## この器がなぜ在るか(先に読め)

**判断は引き継げない。** ユーザーが会話の中で育てた「音」の耳は、メモを読んだだけの新セッションには宿らない。これは Claude の限界で、嘘でごまかさない。

だが引き継げるものが3つある。これが本ファイルの中身:
1. **判断ログ** = ユーザーが一個ずつ直した具体例(✗→✓→理由)。原則は圧縮で意味が抜ける。具体例は抜けない。**解釈するな、パターンマッチしろ。**
2. **gold = `description_legacy`(全490)**。声の手本はここに*物として*在る。記憶でなくこれと並べて照らす。
3. **境界線** = 「ここはお前には判断できない。ユーザーに聞け」。

> 過去の失敗: ハンドオフに「結論」(『』使え 等)だけ書いた → 次セッションはチェックリストとして表面適用 → 照らす相手が無いまま漂流。**結論でなく、この3つを引き継ぐ。**

---

## 唯一の合否 = ヤックン耳テスト

何か作った/直したら、必ず**作る姿勢を捨て、読者の姿勢に座り直して**:
1. 予備知識ゼロの子のつもりで「**うちの出力だけ**」を読む。
2. 次に gold(`description_legacy`)を読む。
3. 並べて確認: **①意味が戻るか ②声が同じか ③機械が漏れてないか**。
4. 一つでもズレたら**未完**。grepゼロ・カバー率・"全部日本語になった" は**偽の頂上**。

## 判断ログ(具体例の山・増やしていく)

### 声・表記(gold=legacyに寄せる。勝手にやわらかくするな)

| ✗ やった | ✓ 正 | 理由 |
|---|---|---|
| `8分の1` | `1/8` | ヤックンは分数スラッシュ表記。勝手に和語化するな |
| `バインド状態` | `『バインド』状態` | 状態名・タイプ名・技名は `『』` で囲む。外すな |
| `あたえる` `にげたり` `あいだ` | `与える` `逃げたり` `間` | 漢字を勝手にひらがなに開くな。ヤックンの漢字使用を保て |
| `はんぶん` `ぜんぶ` | `半分` `全部`(or legacyの語) | 過剰softening禁止。「小学校低学年向け」=全部ひらがな ではない |
| 機械語が出力に: `true` `0.125` `値` `until_user_leaves` `(未対応)` | `交代できない` `最大HPの1/8` `(ラベル無し)` `自分が場を離れるまで` | 機械漏れは出力に出すな(部品→言葉) |

> 核心(ユーザー談): **「全部やわらかい日本語に作り変える」のがゴールではない。** ヤックンの"音"=ヤックンの表記(『』・漢字・1/8)に戻れること。

### 意味(損得・条件・例外を落とすな) — ✓本人確定 5/5(2026-06-06・全行ユーザーの耳でOK)

> 声・表記の表(上)はユーザー自身の言葉＝確定。下はClaudeの聴き分け(paraphrase)で、**ユーザーが言い直して初めて"耳"になる**。それまで暫定。
> **札(provenance) — 訂正(深い罠を1つ潰した)**: 意味(耳)の理由は **Claudeが判定できないカテゴリ**。別のClaudeが「合ってる」と頷いても、paraphraseを承認で量産するだけ(=8分の1の罠が"検証"の衣で一段上に起きる)。だから札は3つ: `✓本人`=人間の耳(**これだけが耳を確定できる**) / `◇機構裏取り`=Claudeがデータ/legacyに錨を打って確認できた**事実・機構のみ**(耳ではない) / `★要`=耳の確定待ち。**Claudeは★→✓に上げられない(別Claudeの同意も不可)。耳は人間のみ。** → ゲート(verifier)も legacy か 人間 にしか錨を打つな。**別Claudeの意見に錨を打つな。**

| 技 | ✗ 生成 | ✓ gold(legacy) | 戻れない理由(★要・本人言い直し) |
|---|---|---|---|
| いばる | 相手のこうげきが2段階**あがる** | 2段階**上げてしまう** | **✓本人(候補OK)**:「損な効果は『〜してしまう』で書く=損だと分かる動詞に。『あがる』だと利益/中立に聞こえ代償が消える」／ 機構=◇データ一致(こんらん付与＋相手attack+2＝自分の代償) |
| はらだいこ | こうげきが最大まであがる | HPを半分減らす**代わりに**最大まで上げる | **✓本人**:「まず自分のHPが最大HPの半分だけ減る → **その代わりに**攻撃が最大まで上がる(順番=HPが先・対価として攻撃)。最大HPの半分なので、**今のHPが半分以下だと失敗**する」／ 機構=◇データ全一致(`HPが減る fr0.5`＋`攻撃to_max`＋`fails_if 現在HP<0.5`) |
| ミストバースト | (条件が)ミストフィールドの時 | …ひこう/ふゆう**など**ではなく**地面にいる時** | **✓本人(候補OK)**:「条件は後半(地面・除外)まで言い切る・接地は『など』を残す(除外リスト不完全)」／ 機構=◇データ一致(condition=field+and:user_grounded+grounded_exceptions。後半を落とすと不完全) |
| のろい | 条件を繰り返してぐちゃつく | 「ゴーストの時は…それ以外は…」と束ねる | **✓本人**:「**条件を先に言う**。自分がゴーストタイプの時だけ、相手をのろい状態(=毎ターン相手のHPを削る)にして、自分はHPを半分払う。ゴーストでなければ相手はのろいにせず、自分のすばやさ-1の代わりに攻撃・防御が上がる。**『相手をのろい状態にする』を先に言わない=条件が先**」／ 機構=◇分岐データ一致。**◇確認済(Bulbapedia＋pokemonwiki/のろい＋本人)**: のろいは失敗しない=HP半分払い、**HP半分以下でも成功し使用者はひんし**・**控えがいなくても成功しバトル敗北**(呪いは相手に入る)。はらだいこ=失敗 / のろい=瀕死、の差。データは`fails_if`無し＝natural f(HPが減る0.5で0になれば瀕死)で既に正しい・要追加なし |
| いのちがけ | 固定ダメージ | 自分の残りHP分のダメージ | **✓本人**:「相手に**自分の残りHP分**のダメージ → 自分はひんし。**ゴーストタイプには無効**・それ以外は**タイプ相性を受けない**を必ず明記(修正せずそのまま)。『固定』は使わない=誤解語」／ 機構=◇`immune:ゴースト`データ済✓(新版が未表示なだけ)・`固定ダメージ`=相性無視は内在→明示化 |

> 上の全行で gold(legacy)の方が上手かった——これが「全部日本語になった」問題。**文法的に正しい日本語 ≠ ヤックンの音。** 生成で作り直す前に、必ず「もう legacy に在るのでは?」を問え。

### 手つき(「足りない」と言う前に裏取り) — 詳細は [[block-review-method]]#7
「欠落／未構造化／immune無し／誤り」と言いかけたら、まず実データで確認。このプロジェクトで実際に3連続誤検出: すなあらし1/16は既に`fraction`で構造化済・とおせんぼうのimmuneは`bd.immune`に在った(effレベルだけ見て「無し」と誤判定＝二層を両方見よ)・いちゃもんkind=ちょうはつは過剰マージ。**「足りない」系の指摘ほど裏取り。多くは"実は既にある／別レベルにある"。**

### 進め方(根本の罠)

- **作る実感 > 確かめる規律 になる**(Goodhart)。英語消し・kind統一・duration正規化・全490生成——全部「測れる手応え」がある。だから機械の方へ流れ、手段を目的と取り違える。
- **既に在るものを壊すな**。生成・言い換えで作り直す前に必ず「**もう在るのでは?**」を問う。`description_legacy` が既に勝っている例が多い(=effectsから生成器を作る前に立ち止まれ)。
- **作る最中のセッションは自分の漂流を感じない**。だから作業の前後で、上のヤックン耳テストを*外から自分に課す*。

## 境界線(ここは判断するな。止まってユーザーに聞け)

セッションは「いま legacy で決まる領域にいるのか / ユーザーの好みでしか決まらない領域にいるのか」を内側から区別できない。両方「ルール適用」で処理して、聞くべき所で確信を持って表面適用する。だから明示する:

- **声の最終判定**(「やわらかくしすぎ」か等)は**ユーザーの耳**。迷ったら出力を見せて聞く。決めるな。
- **legacy / 現行description / effects が食い違う時**: 機構の正は検証済みeffects/現行description、声の手本はlegacy。だが**SSOT内部が矛盾して決着しない時**は、権威ソース(機構なら Bulbapedia 等)で確認 or ユーザーに聞く。記憶で断言するな。(例: フェアリーロックのゴースト除外=Bulbapediaで決着)
- **スキーマを推測で足すな**(新フィールドの発明)。simulator実装と整合する形で、要相談。
- **Champions に何が在るか**等の事実は記憶で断言しない。確認 or 聞く。(例: メガ/Z/ダイマックス/テラスタルは季節制で全搭載=[[champions-battle-systems]])
