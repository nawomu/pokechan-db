# 設計: バトル演出強化スプリント(2026-07-10 阿部さん「音とエフェクトを派手に・じっくり楽しめるように」)

対象: `online_battle.html` / `real_battle.html`(表示層のみ・エンジン`real_battle_simulator.html`のロジック不変)。
回し方: 設計=本書 → 実装=サブエージェント(Sonnet) → 検証=Claude(Playwright実機+スクショ/GIF) → **見た目の最終判定=阿部さんの目**(スクショを都度共有)。夜通し自走OK(阿部さん指示)。

## 原則
- **バトルロジック/ログ文面は変えない**(ロックステップ整合・translateLogLine非破壊)。演出はログ行の解釈(既存fx機構)に足すだけ。
- 音はWebAudio合成(既存`SE`オブジェクトの流儀)。外部音源ファイル禁止(自前合成)。
- 絵はCSS/絵文字/インラインSVG(自前)。公式アセット複製禁止。
- 追加テキストなし(演出のみ)=i18n影響なし。「ミス!」等の文字はログに既にある語を使わず記号/既存語で。
- 両ページ(online/real)に同一実装(表示層はクローン)。エンジンiframeのバスターは触らない(エンジン非改変)。

## 技クラス分類(battle_data/タグから自動判定・優先順)
sound(音技) > punch(パンチ系) > slash(切る) > bullet(弾) > wind(風) > wave(波動) > bite(かみつき) > kick > 物理汎用(体当たり) / 特殊汎用(タイプ色ビーム/波)。
タイプで色付け(TYPE_COLORS)。判定材料: `m.battle_data.flags`・タグ(waza_picker `_miscTagJudges` 相当)・技名パターン(キック/パンチ等)。

## FXプリミティブ(Wave1)
- projectile: 攻撃側→相手へ飛翔体(クラス別の形: 拳/斬線/弾/波リング/音符)。当たったらimpact burst(タイプ色の爆発+軽い画面シェイク+ヒット音)。
- miss: 相手位置で「スカッ」→飛翔体がすり抜けてフェード+ミス音(効果音のみ・文字はログが言う)。
- 特殊技: wave/beam(タイプ色のリング拡散 or ビーム線)。物理: 突進(既存の2段演出を強化)+クラス別プロジェクタイル。

## Wave2: 状態演出
- ランク上昇: 緑の↑粒子が体から立ち上る+上昇スイープ音(ウィーン)。下降: 紫の↓が落ちる+下降音。段数で粒子数変える。
- 回復(たべのこし/オボン/回復技): 緑のキラキラ+ハープ系音。
- 状態異常付与: 既存に色フラッシュ+専用短音(まひ=ビリッ/やけど=ジュッ/こおり=キン/ねむり=Zz/どく=ボコッ)。

## Wave2.5: 接触技=突進モーション(2026-07-10 阿部さん提案・最優先)
- **接触○の技(じゃれつく/パンチ/キック/かみつき等)は飛翔体でなく「体をぶつけに行く」突進**にする(接触=さめはだ/せいでんき等が働く技だから、体当たりのイメージが正しい)。
- 攻撃側スプライトが相手位置へ踏み込み→**接触の瞬間**にクラス別アイコン(👊⚔️🦷)+タイプ色バースト+ヒット音→定位置へ戻る。ミス=相手の手前で空振りして戻る+スカ音。
- 非接触の物理(いわなだれ/じしん等)と特殊は現行の飛翔体/ビームのまま。接触判定はDBの接触フラグ(m.contact系)を使う(技名推測でなくデータ)。

## Wave3: 大物
- メガシンカ: じっくり演出(光柱→シルエット→虹バースト→着地ドーン+専用ファンファーレ和音・計1.5〜2s)。
- 壁: リフレクター=橙半透明パネル/ひかりのかべ=黄 を自陣前に持続表示(エンジンの残ターンはログ・watchパネルから読める範囲で。表示層で貼った/消えたログ行を拾って出し入れ)。
- 天候: 既存表現の強化(あめ=雨粒レイヤー等)は任意。

スクショ/GIFを見せて指摘を反映。音量バランス・テンポ(じっくり感)調整。

## 0. 大原則(全Waveに適用)

### 0-1. 不変の制約
- **エンジン非改変**: `#engine-frame`(real_battle_simulator.html)・ブリッジ `S=w.__sim`・`?v=` キャッシュバスターに一切触れない。
- **ログ文面不変**: battleLog の JA正典文字列・`splitBattleLines`・`translateLogLine`/`battle_log_i18n.js` の PATTERN・`data-bl-src` 保存は不変。演出は `lineWithFx` の**解釈側に足すだけ**。fx判定は必ず `line.src`(JA原文)への正規表現。
- **技術スタック**: CSS(transform/opacity/filterのみ=GPU合成・reflow禁止)+ WebAudio自前合成(tone/noise プリミティブ・外部ファイル禁止・`dest()` 経由・`sndOn` ガード)+ インラインSVG。外部アセット禁止。
- **ロックステップ安全**: 演出は表示のみ。エンジン状態・乱数・同期系(canonSides等)を読まない/変えない。`Math.random` は見た目の散らしのみ可。
- **聖域**: HPバー同期(`setHpBar`/`_hpSnap`/`_idSnap`/`_dgSnap`)・`reorderEndure`・交代の `_recallTimer` の発火順を変えない。
- **i18n**: 演出で足してよい文字は記号/絵文字のみ(追加文字列ゼロ=i18n影響ゼロ)。
- **座標系の使い分け**: 動線系(飛翔体・突進クローン)= `fxPoint()`(getBoundingClientRect)で画面座標を取り **body直下に fixed**(z-index:55)。その場系(バースト・粒子・パネル)= `#f-self`/`#f-opp` 内 append(zoom 1.45 / opp scale(.95) / self scale(1.2) の影響を受ける点に注意=サイズは側で微調整)。

### 0-2. テンポ予算(過剰演出防止)
- **1アクション600ms以内**が上限目安。内訳目安: 発射〜着弾 ≤300ms / 着弾余韻(バースト・シェイク) ≤300ms。**メガシンカのみ特例 1.5〜2.0s**。
- fxは「行が表示される拍」に同期。持続は次の行まで=長い演出は **dur戻り値(`_hitFxDelay`方式)** で後続行を同拍に畳むか1拍内完結。`setTimeout` 遅延演出は次拍と衝突しうるので dur戻り値方式を優先。
- **演出量はダメージ/重要度に比例**(Vlambeer原則)。通常打=控えめ、ばつぐん/急所=増量、KO/メガ=最大。全攻撃で最大演出はインフレ=禁止。
- 音量規律: 新SEは既存 `SE.hit` 以下の音量。密度(レイヤー数)で迫力を出す。

## ② Wave2.5: 接触技=突進モーション(最優先)

### 2-1. 判定(データ駆動)
- `attackFx`(2948行)冒頭で `mv = S.sides[side].moves から技名一致で取得` → **`mv.contact === true`**(WAZA_MAP/battle_data の `"contact":true`)なら chargeFx 分岐。**技名からの推測は禁止**。mv不明時は現行フォールバック(飛翔体)のまま。
- contact でも `moveClassOf` の cls(punch/bite/slicing等)は着弾グリフ・SE選択に引き続き使用。

### 2-2. chargeFx タイムライン(合計 ≤ 520ms)

```
t=0      アンティシペーション: クローンを相手と逆方向へ -10px / 70ms (ease-out)
t=70     突進: fxPoint(自分)→fxPoint(相手)*hitFrac(命中=0.92) へ 160ms
         easing=accel(cubic-bezier(.4,0,1,1))。Y は2段keyframesで低い放物線(ballistic2Under風)
t=230    ★着弾(この時点の経過msを dur として return=既存 _hitFxDelay 同期がそのまま動く)
         同拍: burstFx(タイプ色, cls, intensity) + _RB_CLS_ICON(👊⚔️🦷等クラス絵文字)
              + SE.hitClass(cls, intensity) + fieldShake(mag) + ヒットストップ60-80ms(帰還開始を遅らせる)
              + 被弾側ノックバック(逆方向 12px+rotate±3deg / 60ms → 200ms cubic-bezier(.22,1,.36,1) 復帰)
t=290-310 帰還: 定位置へ 200ms (decel) → クローン削除・本体復帰
```

### 2-3. 実装方式(安全策)
- **本体は動かさずクローンを飛ばす**: `.sprite` を `cloneNode` して body直下 fixed(z-index:55、`.rb-proj` と同レイヤー)に置き、本体は `visibility:hidden`(**display:none禁止**=レイアウト・fxPoint保持)。WAAPI `el.animate` 駆動、`onfinish` で本体復帰+クローンremove。zoom/scale の割り戻し計算が不要になり、`#f-opp scale(.95)/#f-self scale(1.2)` の罠を回避。
- クローン飛行中に交代/ひんし演出(`_recallTimer` 系)が走るケースに備え、**クローンは try/finally 相当で必ずremove**・本体 visibility は必ず復帰(演出例外で盤面が空になる事故防止)。
- ミス: `hitFrac=0.70` の手前で空振り(クローン opacity→.4 で減速フェード)→帰還。burst/シェイクなし、`SE.miss`。dur は空振り時点で return。
- 戻り値契約: **`attackFx` の戻り値 dur = 着弾までの経過ms**(帰還はdur外で流す=「こうかは…」行が着弾と同拍に重なる既存挙動を維持)。

### 2-4. 完了条件(PDCA)
- Playwright実機: 接触技(たいあたり等)で突進・非接触技で従来飛翔体・ミスで空振り、JSエラー0、`_hitFxDelay` 同期(こうか/急所行の同拍表示)非破壊、online_battle.html でも同一挙動+ロックステップ非破壊(観戦側で同演出)。

## ③ Wave3: メガシンカ儀式+壁パネル

### 3-1. megaFx(特例 1.5〜2.0s・「収束→シルエット→爆発」の3拍構造)

メガ行(3245行付近)の `screenFlash('#ffd96b',true)` を `megaFx(side)` に置換。**メガ行の拍を dur戻り値方式で専用ホールド**(`_megaFxDelay` として次行の say を演出完了まで遅らせる=行は跨がない・1拍を伸ばす)。

```
t=0       背景: #fx-flash 系オーバーレイを紫 #835BA5 opacity .6 へ(300ms)+SE.megaロング開始
          (和音アルペジオ triangle 3音上昇+ノイズスウェル lowpass 200→2000Hz を全長にわたり)
t=200     光柱: #f-側に縦グラデdiv(白→透明、transform-origin:bottom、scaleY 0→1 / 250ms)
t=300     収束: 白発光玉(box-shadow glow)×5をポケモン周囲から中心へ scale 2→0.5 / 400ms (decel)
t=500     シルエット化: スプライトに filter: brightness(0) invert(1) drop-shadow(0 0 12px #fff)
t=700     ★スプライト差し替え(renderAll のメガ後画像に更新=変身の瞬間を見せない)
t=900     爆発: 収束玉 scale .5→4+opacity→0 / 300ms、同時に桃色霧玉(radial-gradient桃)×4
          + 虹リング(conic-gradient円+mask輪化、scale 0→5 / 500ms)拡散
t=1000    filter解除(transition 300ms)で新しい姿を披露 + fieldShake(1.6)
t=1300    仕上げ: DNA型メガシンボル(インラインSVG)を中央に120msフラッシュ → 全要素remove
t=1500-2000 終了。megaFx が総dur を返し、次行の say を解放
```

- 全エフェクトは #f-側内 append(側スケールの影響はサイズ係数で吸収)。エンジン状態は読まない(メガ行=JA正典行の検知のみ・既存regex流用)。
- 失敗しても盤面を壊さないよう、remove は timeout の最終ハンドラで一括。

### 3-2. 壁パネル(リフレクター/ひかりのかべ/[オーロラベール])

- **検知**: `lineWithFx` に張った/消えた行の正規表現を追加。**実装前に必ず battleLog の実文言(JA正典)を実機で採取してから regex を書く**(記憶で書かない=文言推測禁止)。エンジン状態は読まずログ行だけで出し入れ(設計書指定)。
- **表示**: `#f-側` の手前(スプライトより前・z上)に平行四辺形div(`transform: skewX(-15deg)`、幅≈100×高≈50px相当)。**持続は #f-側への class付与(`.rb-wall-reflect`/`.rb-wall-screen`)で CSS管理**=再描画(renderAll/交代)に耐える。交代・ひんしでは消さない(壁は側に付く)。消えた行の検知時と新バトル初期化時のみ class除去。
- **色仕様**(Showdown実測準拠): リフレクター=`rgba(238,238,238,.35)`+border 1px `#888`(銀)/ ひかりのかべ=`rgba(220,220,0,.35)`+`#AAAA00`(金)/(将来オーロラベール=`rgba(136,238,255,.35)`)。光沢: `linear-gradient(105deg, transparent 40%, rgba(255,255,255,.6) 50%, transparent 60%)` を重ねてガラス線。
- **出現**: `transform-origin:bottom` scaleY 0→1+opacity .1→.7→.3(400ms+300ms)=一瞬強く光って薄く常駐+壁展開SE(triangle 2音和音・短発)。**消滅**: opacity→0 / 300ms。
- 両ページ反映+観戦側(online)でも同ログ行で同表示になることを実機確認。

## ④ Wave4: juiceパス(磨き)

投入順(効果対コスト降順)。各項は B カタログの仕様に従う。

1. **ヒットストップ本格化**: 着弾瞬間に攻撃クローン+被弾側の WAAPI anim を `anim.pause()`→setTimeout再開(通常60-80ms / KO・急所120-150ms、`duration = clamp(50 + ダメージ%*1.2, 50, 150)`)。背景・パーティクル・BGMは止めない(全停止=バグに見える)。静止中に被弾側へ ±1px ヒット振動。**「止まる→光る→吹き飛ぶ」の順序**が快感の正体(ノックバックはストップ明けに発火)。
2. **インパクトフレーム**: 急所・KO級限定の白黒反転50ms(B仕様)。ヒットストップと同拍発火で衝撃倍増。
3. **トラウマ式シェイク格上げ**: S項の `--shake-mag` 実装を trauma²+sin合成ノイズ+rotate に差し替え(API互換=`fieldShake(mag)` は trauma加算に読み替え)。連続ヒットで揺れが自然に蓄積。
4. **色収差**: 大技(ばつぐん大ダメージ/KO)のシェイクピーク100-150msのみ。
5. **KOスロー+暗転ビネット**: KO行限定(毎回は冗長=とどめだけのご褒美)。
6. **SEマスターコンプ+爆発音**: 全SE最終段コンプで「前に出る」音へ。爆発音はKO・大技限定。
7. **ゴーストHPバー**(条件付き): setHpBar/スナップ機構に一切割り込まない監視レイヤー構成が実機回帰(交代・こらえる・連続ダメージ)で無害と確認できた場合のみ。ダメなら見送り(聖域優先)。

## ⑤ 共通実装基盤(全Waveで共有)

- **プリミティブヘルパー1個に集約**: `showEffect({kind, from, to, color, dur, easing, end})` — kind=particle/ring/glyph/glow/panel、easing語彙は4種のCSS変数(`--ez-linear/--ez-accel(cubic-bezier(.4,0,1,1))/--ez-decel(cubic-bezier(0,0,.2,1))/--ez-swing`)、終端は explode(`scale(2.5)+opacity:0`/100ms)と fade の2種のみ。Showdownが490技を回せる理由=演出のデータ化。以後の技追加を数行にする。
- **タイプ色玉テンプレ**: div+radial-gradient(ほのお `#fff8→#ffdd66→#ff6622→transparent`、みず `#aaddff→#3399ff`、どく `#dd88ee→#7733aa`、ゴースト `#aa88cc→#331144` 等)。色は `mv` のタイプ→既存 TYPE_COLORS 参照、mv不明時フォールバック `#9fb4d8`(現行踏襲)。
- **後始末規律**: 生成ノードは必ず onfinish/timeout で remove(リーク・z-index残骸防止)。演出中の交代/KO割込みでも盤面不変式(本体スプライト可視・パネルclass整合)を守る。
- **検証ゲート(各Wave共通・報告前必須)**: `python3 -m http.server 8000`+Playwright実機で ①JSエラー0 ②対象演出の発火スクショ ③非対象経路(非接触技・通常ダメージ)の無変化 ④`tools/i18n_audit_playwright.js` 残ja=0 ⑤online_battle.html で同演出+ロックステップ同期非破壊(2窓対戦1回) ⑥msg-speed最速設定でも演出が次拍に食い込まない(600ms予算・メガ特例のみホールド)。
