# オンライン対戦 Supabaseセットアップ手順(阿部さん作業ぶん)

方針: Supabase Realtime 一本で「合言葉 + 公開ロビー + ランダムマッチ」の3モード。
まず「合言葉で1対1が繋がる」最小版 → そこにロビー/ランダムを乗せる。
設計の中身は `設計_オンライン対戦_2026-07-07.md`(ホスト権限リレー方式=battleLog差分を送り合う)。

---

## ★あなた(阿部さん)がやる操作(約5分・無料)

### 1. Supabaseアカウントを作る
- https://supabase.com にアクセス →「Start your project」→ **GitHubアカウントでサインイン**(既存のGitHubでOK)。
- ※Claudeはアカウント作成やパスワード入力を代行できない決まりなので、ここはご自身で。

### 2. プロジェクトを作る
- ダッシュボードで「New project」。
- **Name**: 例 `pcham-battle`
- **Database Password**: 適当な強いパスワードを設定 →**メモしておく**(今回のオンライン対戦では実は使わないが、作成に必須)。
- **Region**: **Northeast Asia (Tokyo)** を選ぶ(日本からのレイテンシ最小)。
- 「Create new project」→ 1〜2分で立ち上がる。

### 3. 接続情報(2つ)をコピーする
プロジェクトの **Settings(歯車)→ API** を開く:
- **Project URL**(例 `https://xxxxxxxx.supabase.co`)
- **Project API keys** の **`anon` `public`** キー(長い文字列)

### 4. この2つを私(チャット)に貼る
- **Project URL** と **anon public キー** をチャットに貼ってください。
- この2つはクライアント(ブラウザ)に埋め込む前提の**公開してよい鍵**なので、共有・コミットしてOKです。

### ⚠️ 絶対に渡さない・貼らないもの
- **`service_role` キー**(同じAPI画面にある「secret」表記のもの)= **管理者権限**。これはブラウザにもチャットにも**絶対に出さない**。今回のオンライン対戦(P1〜P3)では**不要**です。
- Database Password も今回は不要(渡さないでOK)。

---

## ★私(Claude)がやること(あなたのキー受領後)

1. **合言葉1v1(P1)**: Supabase Realtimeの「チャンネル(部屋)」で2人を繋ぎ、`team`/`action`/`turnResult` を送り合う配線を real_battle.html に実装。ホスト側だけエンジンを回し、battleLog差分をゲストへ→ゲストは既存の再生機能で表示(演出・多言語そのまま)。
2. **公開ロビー(P2)**: Supabaseの **Presence**(在席表示)で「今オンライン/待機中の一覧」を出す。合言葉なしで一覧から相手を選んで対戦。
3. **ランダムマッチ(P3)**: 待ち行列で先頭2人を自動ペアリング。
4. 各段階ごとに私が実機検証(PC⇔スマホ想定)→ あなたに触ってもらう。

## テスト方法(でき次第)
- あなたのPCで部屋を作る → スマホの別ブラウザで合言葉/ロビーから入る、で1対1が成立するのを確認。

## コスト
- Supabase無料枠(同時200接続・月200万メッセージ)で、身内〜小規模公開まで**実質$0**。中規模500人超で月$25程度(設計メモ§16)。

## 不正(知らない人対戦)について
- P1〜P3は「ホスト権限方式」=片方のブラウザが計算するので、理論上ホストがズル可能。**小規模・半公開なら実用OK**。
- 本格的に公平化するなら将来 **P4=サーバー権限方式**(Supabase Edge Functions等でサーバーが計算)に格上げ。まず動くものを出してから判断。
