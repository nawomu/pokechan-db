# 設計: 連続技(マルチヒット)の演出=回数分・各ヒットでダメージ (2026-07-16)

**発端(阿部さん 2026-07-16)**: 2〜5回連続で当たる技(たねマシンガン/ロックブラスト/スイープビンタ等)は、
回数分モーションを出して相手に一回ずつ当たり、**各ヒットでダメージ数字**を出したい。接触=突進×N・ビーム=ビーム×N・弾=弾×N。
関連仕様として: ①ばけのかわ=連続技の1発目で剥がれ→2発目から普通にダメージ ②相手がサメ肌/ゴツゴツメット=接触1回ごとに自分も反動。

**阿部さんの決定(2026-07-16)**: ①A(連続技)を先にやる ②バトルログの文字は「各ヒットを1行ずつ+最後に合計」。

**体制**: 設計/検証/commit=Opus・実装=Sonnet。★real_battle.html/online_battle.htmlに触るので、同時期に同ファイルを触る他タスク(glyph等)とは**並列にしない**(2026-07-13教訓)。sim=バトル土台なので**回帰(sim 823/825ベースライン)必須**+権威(Bulbapedia/ポケモンWiki)整合。

## §1 現状(調査で確定・file:line)

### sim(real_battle_simulator.html)
- `phaseDealDamage`(2552〜)の連続攻撃ループ(2759〜2790): `for(h=0..hitCount)` で各ヒットのダメージ`v`を計算し`total`に積む。**ログは1行だけ**: `〇〇の 技！ N回ヒット 合計X ダメージ！ (残HP h/max)`(2796)。単発は`〇〇の 技！ 対象に X ダメージ！ (残HP …)`(2797)。
- `hitCount`決定(2728〜2748): `hits`固定 / `min_hits`〜`max_hits`(2〜5は35/35/15/15%・スキルリンク=最大)/ stop_on_miss(トリプルアクセル等)/ hits_by(ふくろだたき)/ おやこあい2発。
- **ばけのかわ**(1567 calcDamage内・**ループの前**): `def.disguise && move.category!=='変化'` で `immune:true, disguiseBlock:true` を返し**技全体を0ダメージでブロック**。→ 連続技でも1発で全ブロック=阿部さん要望と違う(1発目だけブロックすべき)。
- **さめはだ/てつのトゲ**(2839〜): ループの**外で1回だけ** `1/8` 反動をログ。→ 連続技で接触ごとになっていない。
- **ゴツゴツメット反動**(helmetRecoil=`realStat/6`・calcDamage 2104で1回算出→2960でループ外で1回減算)。→ 同上、接触ごとでない。
  - ★権威: Bulbapedia "Disguise"(マルチヒットは1発目を防ぎ以降ヒット)/"Rough Skin"・"Rocky Helmet"(接触ヒットごとに発動=マルチヒットは回数分)。

### 表示(real_battle.html / online_battle.html)
- `lineWithFx(msg, opts)`(3287〜): 行が `^X の Y！` かつ `ダメージ！`(かつ`で N ダメージ`でない)なら**攻撃cueを1回再生**。ダメージ数=`/(?:に|合計) (\d+) ダメージ/`。残HP=`/\(残HP h(/max)?\)/`で**その行でバーを減らす**(3305)。`resolveCueSheet(mv)`→`playCueSheet`(cueあり)or `attackFx`(cueなし)。
- 呼び出し元(3048付近): `splitBattleLines(e.msg).forEach(t => lines.push(lineWithFx(t, {...})))` = **log entry `e` が手元にある**(→ `e.multiHit`等のフラグを`opts`へ渡せる)。
- 現状の連続技: summary1行しか来ないので**演出1回・合計ダメージ1個**だけ。

## §2 実装方針

### 2-1. sim: 各ヒットを1行ずつ出す(+接触反応を毎ヒット化)
連続ループ(2759〜)を改修:
1. 各ヒット`h`で、**単発と同じダメージ行フォーマット**を出す(i18n流用=新パターン不要): `〇〇の 技！ 対象に ${dh} ダメージ！ (残HP ${def.currentHp}/${result.hp})`。
   - ★log entryに **`multiHit:true`(+ `hitIndex:h`, `hitCount`)** を付ける(表示側が「速い連打演出」に切替える信号)。`log()`がentryにメタを載せられるか確認し、載せられなければ`log()`を薄く拡張(第3引数meta・省略時=従来)。
2. **ばけのかわを1発目だけに**: disguiseブロック判定を**ループ内h===0へ移設**(1発目=ダメージ0+「ばけのかわが はがれた！」+`def.disguise=false; def.disguiseBroken=true`)。2発目以降は通常ダメージ。単発技(hitCount===1)は従来と同結果(1発でブロック=不変)。※calcDamage側のdisguiseBlock分岐との整合を保つ(二重防止)。
3. **さめはだ/てつのトゲ/ゴツゴツメット反動を接触ヒットごと**に: 接触技かつ実ダメージが通ったヒットごとに`1/8`(さめはだ/てつのトゲ)・`1/6`(ゴツメ)を反動。ループ内で毎回、または「実接触ヒット数」を数えてループ後にまとめて(見た目の反動ログは合計1行でも可=阿部さんは「一回当てるごとに食らう」と表現→ヒット数分の合計ダメージが要点)。★実装は「接触ヒット数×単位反動」を攻撃側HPから減算(0未満クランプ・ひんし判定)。マジックガード/パンチグローブ/マジックルームの既存ガードは維持。
4. **summary行**(阿部さん「最後に合計」): 攻撃fxを誘発しない形式にする(`^X の Y！`で始めない)。例: `${hits}回 あたった！(合計 ${total} ダメージ)`。→ **新i18nパターン1つ**(9言語)。残HPは各ヒット行で反映済みなのでsummaryに残HP不要(付けるなら表示は最終バーの再確認のみ)。
5. 途中でひんし(`def.currentHp<=0`)なら残りヒット中断(既存`break`維持)=それまでのヒット行だけ出る。

### 2-2. 表示: 連続技は「速い連打」で再生
`lineWithFx`で`opts.multiHit`の時:
- 攻撃cueを**圧縮した速いストライク**で再生(full 2.3sを回数分は長すぎる=5発11秒はNG)。案: multiHit時は`playCueSheet`に**compress係数**を渡し、charge/projectile/beamのatk尺とburst/popnumを短縮(1発 ≈ 350〜500ms目安)。または専用の軽量cue(quick strike)を用意。**各ヒットでダメージポップ(その発のdh)** を出す。バーは各行の残HPで段階的に減る(既存3305)。
- `say()`の送り(メッセージ送り速度)を**連続ヒット行だけ速く**する(`hold`短縮=W3で入れた`line.hold`機構を流用)ので、パッパッと連打に見える。
- 最終ヒットの後に従来の「こうかは…/きゅうしょ」ビートが乗る(既存フロー維持)。
- ★接触=突進クローンを毎ヒット出す時、前ヒットのクローン掃除(cleanup)と次ヒット開始が競合しないよう順序を担保(連続再生の重なりに注意=W4の`__FX_SCRUB__`とは別・通常再生のcleanupタイミング)。

### 2-3. i18n
- 追加は**summary新パターン1つ**(`N回 あたった！(合計X ダメージ)`)を`battle_log_i18n.js`のPATTERNS/TPLに9言語(JA正典→translateLogLineパターン。[[i18n-static-and-battlelog-arch]])。並列WFで7言語翻訳。
- 各ヒット行は単発ダメージ行の既存パターンを流用=新規翻訳不要。
- さめはだ/ゴツメ反動の合計ログは既存パターン流用(文言不変)。
