## 📐 能力ポイント振り分け 公式ルール

**出典** (2026-05-15 複数ソースで横断確認):
- https://gamewith.jp/pokemon-champions/546461 — **赤縦棒=+2 / 黄縦棒=+1 を明記**、UI 内色分けの公式表記
- https://gamerch.com/pokemonchampions/970094 — 「色の濃い目盛り = +1 ボーナス」
- https://yakkun.com/ch/changes.htm — SP (能力ポイント) は性格補正の **内側** に加算される計算順
- https://x.com/takaya7s/status/2027587949597233527 — 計算式 `(種族値+20+SP)×性格補正` の切り捨て

### マイナス補正 (×0.9) の挙動について
**公式ドキュメントおよび UI ではほぼ言及なし** だが、計算式 `floor((inner+pts)×0.9)` から **数学的必然として +0 になる pt 位置は必ず発生**する。約 10pt 毎に 1 回、振っても実数値が上がらない pt が出る。
- party_checker は独自に **青ライン** でこの位置を可視化中（公式 UI にはない可能性が高い）
- 計算式自体は公式と同一なので、表示するかどうかは UX 判断

### 数値ルール
| 項目 | 値 |
|---|---|
| 各ステ最大 pt | **32** |
| 合計 pt 上限 | **66** |
| 通常 1 pt あたりの実数値増加 | **+1** |
| 性格 ↑ ステに 1 pt 振った時 | 基本 **+1**、目盛りが色濃い位置に当てると **+2**（10pt毎付近で発生） |
| 性格 ↓ ステに 1 pt 振った時 | 基本 **+1**、目盛りが色濃い位置に当てると **+0**（10pt毎付近で発生・"無駄" pt） |
| 中性 (性格補正なし) | 常に **+1** |
| Lv | 50 固定 |
| 個体値 (IV) | 31 (V) 固定 — Champions は最初から全 6V |

### 数学的解釈 (公式と等価)
過去作の Pokemon: 性格補正は計算の最後にまとめて適用
```
actual = floor(((base*2 + IV + EV/4)/2 + 5) * nature)
```

Champions: 性格補正は **(内部値 + pts) 全体** に適用 (段階的反映)
```
actual = floor((inner + pts) * nature)
  where inner = floor((base*2 + 31)/2) + 5  (HP以外)
        inner = floor((base*2 + 31)/2) + 60 (HP)  ← 性格非対象なので × nature 省略
```

この式により ×1.1 ステでは 1pt = 実数値 +1.1 が「実数値整数」に丸められ、
**10 pt 経過ごとに +2 増加するタイミング = 色濃い目盛り** となる。

### 検証 (公式攻略サイトと一致確認済み)
| ポケモン | ステ | base | 性格 | pts | 実数値 | 確認 |
|---|---|---|---|---|---|---|
| カイリュー | atk | 134 | いじっぱり↑ | 25 | 196 | ✓ |
| カイリュー | atk | 134 | いじっぱり↑ | 26 | 198 | ✓ (+2 = 濃い目盛り) |
| メガピクシー | def | 93 | ずぶとい↑ | 0 | 124 | ✓ |
| メガピクシー | def | 93 | ずぶとい↑ | 32 | 159 | ✓ (旧 EV252 と一致) |

`floor((154+25)*1.1) = floor(196.9) = 196` のように整数丸めで再現可能。

### バトルシミュレータ実装時の注意
- 計算順は厳密に **「内部値 + pts → ×nature → floor」**
- pts は EV/4 ではないので EV ベースの旧コードを流用する場合は要書換え
- HP は性格補正対象外なので `inner + 60 + pts` で OK
- 個体値 (IV) は 31 固定で良い (Champions の仕様)

## 🎯 何を実装したか (2026-05-14〜2026-05-15)

### A. ステータス調整ポップアップ (Champions 完全再現)
party_checker のステータス表で **V6 セルをクリック→ポップアップ** で能力ポイントを割り振れる機能。

**仕様**:
- 各ステータス 0–32 pt / 合計上限 66 pt
- 性格補正は段階反映 (上記公式ルール参照)
- Lv50・IV=31 固定
- **行内バー直接操作型**: 各ステ行の 32 分割ゲージをクリック / ドラッグで pt 設定
- 性格 ↑ ステは bar 内の 10pt 毎の位置に「濃い目盛り」を視覚化
- 旧右パネル（◀ ▶ MIN MAX）は廃止（ゲージ直接操作で十分）
- キーボード: ← → で ±1、↑ ↓ で行移動、Esc で閉じる

**計算式 (party_checker.html 内 computeActualStat)**:
```js
function computeActualStat(base, statKey, pts, natureName) {
  if (statKey === 'hp') return Math.floor((base * 2 + 31) / 2) + 60 + pts;
  const inner = Math.floor((base * 2 + 31) / 2) + 5;
  return Math.floor((inner + pts) * getNatureMul(natureName, statKey));  // 性格は (内部値+pts) 全体に適用
}
```

**UI**: 単一カラム (旧 2 カラム廃止)
- 6 行のステ表示: アイコン / ラベル(+性格↑↓矢印) / 実数値 / **直接操作バー** / pt 数
- バー直接クリック / ドラッグで pt 設定 (0-32)
- 性格↑ ステはバー内の 10pt 毎の位置に「濃い目盛り」を視覚化 (CSS gradient で 31.25% / 62.5% / 93.75%)
- 下部: 性格ドロップダウン + 効果テキスト（例「ぼうぎょ↑ こうげき↓」）

**キーボード**: `← →` で ±1、`↑ ↓` で行移動、`Esc` で閉じる
**タッチ対応**: iPad / iPhone で touchstart→touchmove でドラッグ可能

### B. 性格選択 5×5 グリッドポップアップ
性格セルクリックで Champions の本家と同じ 5×5 グリッド表が開く。
- 行 = 上がるステ、列 = 下がるステ
- 対角線上が中性5種（まじめ・どんかん・てれや・きまぐれ・がんばりや）
- 黄色ハイライトで現在選択中
- 各セルホバーで「○○↑ ○○↓」プレビューを下部に表示

### C. V6 セルが「現在の実数値」表示に変化
従来は V6 = IV31・EV0・補正なし の固定値だったが、現在は **性格と pts を反映した実数値** を表示。
- pt=0 + 中性: そのまま (例: 113)
- pt=0 + ずぶとい (Def+): 124（+11）
- pt=32 MAX + ずぶとい: 156

### D. メイン表の TD 全体クリック対応
ステセル内のどこをクリックしてもポップアップが開く（旧仕様: 種族値スパンのみクリック可）。

### E. 配色 (オレンジ/赤の混同問題を回避)
| 状態 | 色 |
|---|---|
| pt > 0 (未MAX) | 薄黄 `#ffe066` |
| pt = 32 MAX | ピンクマゼンタ `#ff5ee0` |
| 性格↑ | 赤珊瑚 `#ff7b72` |
| 性格↓ | 青 `#79c0ff` |
| バーグラデ | 薄黄 → オレンジ → ピンク |

### F. PC タブとの統合
`partyNatures` と `partyEvs` は **タブごとに保存**（`captureCurrentPcState` / `applyPcState` に組み込み済み）。

### G. 雑多な周辺修正
- ポケモン選択モーダルに **右上 × ボタン** (`#modal-close-x`) — スマホで閉じやすい
- `#main-wrap` の自前スクロールバーを完全非表示（横スクロールはミラーバー1本のみ）
- 古い「→実数値」バッジは除去（列ずれの原因）
