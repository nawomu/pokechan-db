### 1-2. POKEMON_LIST の必須フィールド(シムが実際にアクセスするもの)

実コード上で参照確認済み:

| フィールド | 用途 |
|---|---|
| `name` | ポケモン名・ログ表示・逆引きキー |
| `type1` / `type2` | タイプ相性・STAB判定 |
| `hp` / `atk` / `def` / `spatk` / `spdef` / `spd` | 種族値 |
| `ab1` / `ab2` / `ab3` | 特性 |
| `weight_kg` | 重さ依存技(けたぐり等) |

| `form` | フォーム名・メガシンカ判定 |
| `resist` (配列) | タイプ耐性プリコンピュート |
| `total` | 表示用 |

シム内で**動的に付与される**フィールド(DBに不要): `gender`・`effort`・`nature`・`item`・`moves`・`currentHp`・`status` 等。

### 1-3. WAZA_MAP の必須フィールド

| フィールド | 用途 |
|---|---|
| `name` | 技名・ログ |
| `key` | 逆引き・learners結合 |
| `type` | タイプ |
| `category` | 物理/特殊/変化 |
| `power` | 基本威力 |

### 1-4. isPickable ロジック(real_battle.html より)

```
isPickable = p => !p.mega && !BATTLE_ONLY_FORMS.has(p.name)
BATTLE_ONLY_FORMS = {イルカマン(マイティ), ギルガルド(ブレード)}
```

全国版では「変身でしか出ない姿」が増える(メモリ: transform-forms-not-startable)。  
→ BATTLE_ONLY_FORMS の拡張が必要。全国版ビルダー(build_national_view.js)で除外済のコスメティックフォームとは別管理。

## 2. 全国版DB(pokechan_data_all.js)との差分(フィールド比較)

### 2-1. POKEMON_LIST のフィールド差

| フィールド | Champions pokechan_data.js | National pokechan_data_all.js | sim への影響 |
|---|---|---|---|
| 種族値/タイプ/特性 | あり | あり | 問題なし |
| `weight_kg` | あり | あり | 問題なし |
| `resist` | あり(プリコンピュート) | あり | 問題なし |
| `mega` / `form` | あり | あり | 問題なし |
| `gen` / `season` | なし(Champion版) / あり(生成) | あり | simは不要(選択フィルタのみ) |
| `legend` | なし | あり | simは不要 |
| 技の習得フラグ群(100+列) | **あり**(champions版の特徴) | **なし** | 重要: 下記参照 |
| `isPickable` | なし(runtime計算) | なし(runtime計算) | runtime計算で対応 |
| `gender` | なし(UI設定) | なし | 問題なし |

**重要な差**: Champions版 POKEMON_LIST には各技の習得フラグ(`コウモリがえし: true` 等の100+列)が含まれる。  
全国版 POKEMON_LIST にはこれらのフラグは**ない**。代わりに `POKEMON_WAZA` (ポケモン名→技slugリスト) と WAZA_MAP の `learners`[] で習得関係を持つ。

→ real_battle_simulator.html のusableMoves関数:  
```js
return Object.values(WAZA_MAP).filter(m => Array.isArray(m.learners) && m.learners.includes(poke.name));
```
これは `learners` ベースなので**全国版でも動く**。Champions版の特殊フラグ列は不要。

### 2-2. WAZA_MAP のフィールド差

| フィールド | Champions | National | 差異 |
|---|---|---|---|
| 基本フィールド(name/type/category等) | あり | あり | 同一 |
| `battle_data` | あり(全498技) | あり(全920技) | 同一 |
| `battle_data.effects` | 470/498 non-empty | 867/920 non-empty | 下記参照 |
| `priority` | あり | あり(全920技) | 全国版のみ全数あり |
| `learners` | あり | あり(822/920技) | 全国版は98技がlearners空 |
| `national_new` | なし | あり(boolean) | simは無視してよい |
| `availability` | なし | あり | simは無視してよい |
| `subcategory` / `tags` | なし | あり | simは無視してよい |

### 2-3. ABILITY_DESC の差

| 項目 | 状況 |
|---|---|
| Champions ABILITY_DESC | 192件 |
| National ABILITY_DESC | 192件(同一・build_national_view.jsがpokechan_data.jsから転記) |
| 全国版ユニーク特性数 | 305種 |
| DESC未設定の特性 | 119種(未対応) |

→ 全国版simで新しいポケモンを使うと特性説明ポップアップが空になる。  
優先度は低い(simの動作には影響しない)が、UXのために追加が望ましい。

## 5. UI設計: real_battle.html との関係

### 5-1. 選択肢の比較

| 案 | 概要 | メリット | デメリット |
|---|---|---|---|
| **A: 母集団切替(推奨)** | real_battle.html に「データソース切替スイッチ」を追加し、pokechan_data.js ↔ pokechan_data_all.js を実行時に切り替える | 既存UIを壊さない・R2クローン方式に準拠・1ページで完結 | データ読込が重くなる可能性(pokechan_data_all.jsは約4.5MB) |
| **B: 別ページ** | real_battle_national.html を新設(real_battle.htmlのクローン + data source差替) | 本番を汚染しない・段階テストが容易 | URLが増える・管理コスト |
| **C: real_battle_simulator.html 拡張のみ** | 現行の検証用simにデータ切替を追加 | 実装最小 | UIがShowdown風でなく本番体験と乖離 |

**推奨: 案B(別ページ方式)でPhase 1を実施**。  
理由:
- R2(クローン方式)に完全準拠: `<script src="pokechan_data_all.js">` の1行差替。
- real_battle.html 本番(Champions)を汚染しない → Championsシーズン運用が続く間、本番を安全に保つ。
- テスト合格後に case Aへの統合(スイッチ追加)も容易。
- pokechan_data_all.js の重さ問題: 現状4.3MB(未gzip)。GitHub Pages はgzip配信するので実用上問題ないが、ページ上に表示するポケモン数フィルタ(gen / 最初の一部等)を検討。

### 5-2. real_battle.html に必要な改修(全国版対応時)

1. `<script src="pokechan_data_all.js">` に差替(クローン方式・CSS/JS変更なし)
2. `BATTLE_ONLY_FORMS` セットへの追加が必要な全国版フォーム:
   - イルカマン(マイティ)・ギルガルド(ブレード) は既存
   - 全国版で追加が必要な変身形フォームを要洗い出し(量:十数件程度)
3. **ポケモン選択UIの絞り込み**: 1219件はスクロール量が多い。Genフィルタ or タイプフィルタの追加が望ましい。
4. `gender` フィールド: 全国版の master_pokemon.json には gender_rate が含まれない → UIでの手動設定(現行と同じ)で問題なし。

> 決める点: 「全国版simは別ページ(`real_battle_national.html`)でよいか、real_battle.html内のスイッチにするか」

### 判断B: 全国版simの対象範囲(フェーズ1前)

テスト工数の観点で段階的にしてもよい。

> 決める点: 「Phase 1で一気に1219体にするか、Genフィルタ付きで段階的にするか」

### 判断C: G-Max技の取り扱い(フェーズ1〜2)

G-Max33技は現在未収録(保留判断・HANDOFF_2026_07_04)。

simエンジン側の対応がなければ「ダメージのみ」扱いになる。  
「G-Maxを選択肢に出すが挙動は近似のみ」か「G-Maxをlearnerから除外して出さない」か。

> 決める点: 「全国版simでG-Max技を見せるか隠すか」

### 判断D: UX — ポケモン選択UIの件数対策(フェーズ1〜4)
