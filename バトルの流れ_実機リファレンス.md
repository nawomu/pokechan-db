# ポケモン バトルのフェーズ構造 — 確定リファレンス

最終更新: 2026-06-08 JST / 統合担当作成
対象世代: 第8世代(剣盾)〜第9世代(SV)〜ポケモンチャンピオンズ(計算構造は同一)

---

## ★大原則(この文書の出発点)

**ポケモンのゲーム(バトル)が出発点である。** `effects`(`pokechan_data.js` の `WAZA_MAP[key].battle_data.effects`)は、バトルシミュレーターが読み取る前提で設計された**機械向けデータ**であり、説明文(`compose`=`tools/_waza_compose.js`)はそのバトル挙動を**ただ日本語に訳しているだけ**である。

したがって、技を解析するときも・effects を足すときも・説明文を点検するときも、**常に「バトルの流れ=フェーズ」を基準に考える**。「いつ起きるか(優先度フェーズか/攻撃後の追加効果か/ターン終了か)」をまず確定し、それから effects のキーと説明文の文言を決める。フェーズを無視して文言だけ直すと、必ずどこかで意味がずれる。

> このリファレンスは、実機の処理順(本編の事実)・我々の sim「Phase構造 v7」・effects への phase 付与運用の三者を**1本に突き合わせた**もの。実機の事実は外部権威ソース、sim 実装状況は `real_battle_simulator.html` 実コードを根拠とする。

## 1. 1ターンの大きな流れ(マクロ・フェーズ)

ターン全体は次の段階で進む。確信度: **高**(複数ソース一致)。

| 段階 | 内容 | 我々sim Phase v7 ラベル |
|---|---|---|
| ① 行動選択 | 両者が 技/交代/道具/逃げる を同時・非公開で選ぶ | (なし=UI) |
| ② 交代・道具・逃げる | 最初に処理。**交代は他のどの行動よりも先**。ただし相手が「おいうち」を選んでいた場合は、おいうちが交代前に2倍威力で割り込む | (未実装) |
| ③ メガシンカ / ダイマックス / テラスタル | 交代の**後**、技フェーズの**直前**。テラスタルはメガ/ダイマの直後 | `Init-B`(未実装) |
| ④ 技フェーズ | 優先度 → 素早さ → トリックルームの順で各ポケモンが技を実行(後述 §2・§3) | `Pre`/`0a`/`0b`/`0c`/`Atk-Base`〜`9` |
| ⑤ ターン終了処理(残留効果) | 天候・状態異常スリップ・やどりぎ・バインド継続などを決まった順で処理(後述 §5) | `10a`〜`10h` |

**戦闘開始時(ターン①の前・1回だけ)**: 場に出たポケモンの特性が**速度順に発動**(いかく/天候特性/フィールド特性など)。我々sim では `Init-A`。

## 2. 技フェーズ — 誰が先に動くか(行動順の決定)

同ターン内で技を出す順は、上から順に判定する。確信度: **高**。

1. **技の優先度ブラケット**(+5〜-7。高いほど先)
2. **道具/特性による同ブラケット内割り込み**: せんせいのツメ・イバンのみ・クイックドロウ(先制側)
3. **行動を遅らせる効果**: こうこうのしっぽ・まんぷくおこう・特性「あとだし(Stall)」(後攻側)
4. **補正込みの素早さ**(高い順。**トリックルーム時は逆順=遅い方が先**。優先度ブラケットは反転しない)
5. 素早さ同値なら**乱数**

**第8世代以降の重要点(動的素早さ)**: ターン中の素早さ変化・優先度変化が**即座に反映**される(第7世代までは行動順がターン開始時固定)。我々sim はこれを満たす(`effectiveSpeed()` がランク・状態異常・持ち物を都度計算)。

## 3. 1つの技を実行するときの内部手順(★effects の phase 付与の土台)

各ポケモンが自分の番に技を出すとき、おおよそ次の順で処理される。確信度: **中〜高**(大枠は各源一致、細部は粒度差あり)。

要件4に沿って、**技1つの実行フェーズ**を次の5段に整理する。これが effects へ phase を持たせる際の基準フレームである。

| 順 | 実行フェーズ | 内容 | ここに入る effects/効果の種類 |
|---|---|---|---|
| 1 | **優先度** | 行動順の確定(§2)。技固有の優先度値 | `priority`(数値)。先制/後攻。優先度を変える特性条件 |
| 2 | **攻撃前の変化** | 行動可否判定(ひるみ・まひ・こおり・ねむり・こんらん自傷・かなしばり等)→ 溜め/半無敵の開始・解除 → 命中判定(命中率×回避率。必中・ロックオンはスキップ) | 使用条件(`requires`=ねむり時のみ等)/ 溜め技の溜めターン能力上昇(ロケットずつき・メテオビーム等)/ 自己強化弱化のうち**変化技そのもの**(つるぎのまい・からをやぶる)/ 必中(`ignores` 回避率上昇・命中率低下)/ まもり貫通・みがわり貫通の判定 |
| 3 | **攻撃** | 急所判定 → ダメージ計算(§4)→ ダメージ適用 → こらえる/がんじょう/タスキ/ばけのかわ等の耐え処理 | 威力(`power`)・タイプ・物理/特殊・複数対象・急所段階・威力倍率系(てつのこぶし等) |
| 4 | **攻撃後[適用後の追加効果]** | secondary effect。**相手のHPが残った時のみ発動** | 状態異常付与(`状態付与`)/ 能力ランク変化(相手↓・自分↑↓ともここ。インファイト自分防御-1・りゅうせいぐん自分とくこう-2 も**攻撃後**)/ ひるみ / 反動ダメージ(すてみタックル)/ HP吸収 / 接触特性・防御側きのみ・いのちのたま反動 |
| 5 | **ターン終了/持続** | この技がターン終了フェーズに残す継続効果 | バインド継続(まきつく)/ あくび(次ターン眠り予約)/ ねがいごと・みらいよち(遅延)/ ほろびのうたカウント / 場・天候・壁・ルームのカウント / recharge(はかいこうせん次ターン)/ ロック(あばれる・げきりん) |

### ★運用提案: effects に phase を明示する(「なし」も明示)

各 effect に `phase` を持たせ、上記5段(あるいは v7 の細粒度ラベル `0a`/`0c`/`9`/`10a`〜`10h`)のどこで処理されるかを必ず指定する。**該当フェーズに効果が無い技は、空のまま放置せず「なし」を明示**する運用を推奨(=「攻撃後の追加効果なし」と「未調査」を区別できる)。これにより compose は「攻撃前に何を言うか/攻撃後に何を言うか/ターン終了に何を言うか」を phase で機械的に並べられる。

> 例(いびき・2026-06-07発見の取りこぼし): ①使用条件=**攻撃前**(「自分が『ねむり』のときだけ使える」)②音系・みがわり貫通=判定は**攻撃前**だが文言は技性質として付記 ③ひるみ30%=**攻撃後**。フェーズで分ければ compose がどれも落とさない。

### ちからずく等の例外(phase 設計上の注意)
**ちからずく**は「攻撃後[適用後の追加効果]」フェーズの効果を**まるごとスキップ**して威力1.3倍に変換する仕様。対象=相手の状態異常/こんらん/ひるみ・相手の能力ダウン・自分の能力アップ・場の変化。**対象外**=反動・**自分の能力ランクを下げる効果**(りゅうせいぐん/インファイト)・ダイマックス技の追加効果。→ phase=「攻撃後」の effect には「ちからずく対象か否か」の区別が要る。確信度: **高**。

出典: [The Cave of Dragonflies](https://www.dragonflycave.com/battle.aspx)(命中→急所→ダメージ→secondary)、[じゃらの箱: ダメージ・回復の順番](https://jara-blog.com/damage-order)、[ポケモンWiki: ちからずく](https://wiki.pokemonwiki.com/wiki/%E3%81%A1%E3%81%8B%E3%82%89%E3%81%9A%E3%81%8F)、[ポケモンWiki: 溜め技](https://wiki.pokemonwiki.com/wiki/%E6%BA%9C%E3%82%81%E6%8A%80)、[じゃらの箱: ひるみ](https://jara-blog.com/hirumi_sv)

## 4. ダメージ計算の内部順(§3の「攻撃」フェーズの中身)

第5世代以降〜第9世代で計算構造は共通。基礎式(Bulbapedia、乗算は左から順・各乗算ごとに丸め):

```
ダメージ = ((2×レベル÷5+2) × 威力 × A ÷ D ÷ 50 + 2)
          × Targets × PB × Weather × GlaiveRush × Critical
          × random × STAB × Type × Burn × other × ZMove × TeraShield
```
- `A`=補正後の攻撃実数値、`D`=補正後の防御実数値。急所時は A はマイナスランク無視・D はプラスランク無視で再計算。

### 端数処理の使い分け(★最重要)
- **基礎ダメージ部の各除算**: その都度**切り捨て**
- **基礎ダメージ以降の各補正**: 原則**五捨五超入**(0.5ちょうどは切り捨て・0.5超は切り上げ。Bulbapedia "rounding down at 0.5")
- **例外で切り捨て**: **乱数**適用後・**タイプ相性**適用後
- **固定小数点4096(Q12)方式**: 1.5倍=6144 / 0.5倍=2048 / 0.75倍=3072 / 2倍=8192 / 1.3倍=5325(割り切れず量子化誤差) / 1.2倍=4915。最終ダメージ最小値は1。

### 日本語Wikiの「8ブロック」構造との対応(我々sim の Phase 1〜8 の元ネタ)
確信度: **中**(出典は番号を明示せず、ブロック構造から再構成)。

| 我々sim Phase | 内容 | 丸め |
|---|---|---|
| Atk-Base | 攻撃実数値(素+ランク、はりきり等) | — |
| 1 | 威力補正値(特性/天候/フィールド/持ち物/条件) | 五捨五超入 |
| 2 | 最終威力 | 五捨五超入 |
| 3 | 攻撃補正値(こんじょう/ハチマキ/わざわい等) | 五捨五超入 |
| 4 | 最終攻撃 | — |
| Def-Base | 防御実数値(素+ランク、急所時プラスランク無視) | — |
| 5 | 防御補正値(ファーコート/しんかのきせき等) | 五捨五超入 |
| 6 | 最終防御 | — |
| 7 | ダメージ補正(複数対象0.75・急所1.5・天候・やけど物理0.5・壁) | 五捨五超入 |
| 8 | 最終ダメージ(乱数→STAB→タイプ相性→other:いのちのたま等) | 乱数・相性は切り捨て、他は五捨五超入 |

検証例(pokewiki): Lv50・こうげき182ガブの威力100じしんが ぼうぎょ100相手に → 基礎82 → 乱数69〜82 → STAB1.5で103〜123 → 相性0.5で**51〜61**。

出典: [育成考察Wiki: ダメージ計算式](https://latest.pokewiki.net/%E3%83%80%E3%83%A1%E3%83%BC%E3%82%B8%E8%A8%88%E7%AE%97%E5%BC%8F)、[Bulbapedia: Damage](https://bulbapedia.bulbagarden.net/wiki/Damage)、[Smogon: Complete Damage Formula for B&W](https://www.smogon.com/bw/articles/bw_complete_damage_formula)、[ポケチャンWiki: ダメージ計算式](https://champions.pokewiki.net/%E3%83%80%E3%83%A1%E3%83%BC%E3%82%B8%E8%A8%88%E7%AE%97%E5%BC%8F)
