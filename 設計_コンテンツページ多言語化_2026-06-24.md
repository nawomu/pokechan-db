# 設計案 — コンテンツページ(pokemon/ ability/ type/)の多言語化

作成: 2026-06-24(実装前の設計レビュー用)。承認後に `tools/_gen_content_pages.js` を改修して再生成。

## 対象と現状
- `/ability/<特性名>.html`(193) `/pokemon/<slug>.html`(276) `/type/<タイプ>.html`(18) + 各index = **約487の静的HTML**。
- すべて `tools/_gen_content_pages.js` が `pokechan_data.js` から**自動生成**(テンプレ共通)。
- 現状 `<html lang="ja">`・ベタ書きja・runtime.js も data-i18n も無し・i18n監査ハーネスの対象外。
- データ在庫: i18n辞書に 特性名+効果(200/200)・タイプ名・分類(genera)・ポケ名(315) すべて揃っている。特性効果の非jaは**PokeAPI short_effect**、jaは独自(=[[i18n-multilingual-rule]]の方針どおり)。

---

## ★最大の論点: SEOアーキテクチャ(2案)

### 案A: 言語別の静的URL(SEO最適・推奨)
- 生成物を言語ごとに分ける:
  - ja(現状維持): `/ability/アイスボディ.html`
  - 非ja: `/en/ability/ice-body.html` `/ko/ability/...` …(英語slugで統一・SEOに有利)
- 各ページ: `<html lang="en">`・**翻訳済み title/description/og**・本文翻訳・`<link rel="alternate" hreflang="...">` で全言語版を相互リンク + x-default。
- sitemap も言語別 or hreflang注記。
- **規模: 487 × 9言語 ≈ 4,400ページ**(生成器のループに言語次元を足すだけ。出力は大きいがテキストで管理可)。
- ✅ **メリット**: Googleが言語ごとに別URLとして索引=海外オーガニック流入の入口になる(=今回「コンテンツ先・SEO効果大」を選んだ狙いに直結)。各言語で正しい title/desc が出る。
- ⚠️ コスト: 生成器の中規模改修・リポジトリにファイル4400・sitemap言語対応。

### 案B: 単一URL + クライアントi18n(簡単だがSEO弱)
- テンプレに runtime.js + data-i18n + DB参照描画を足す。URLは1エンティティ1つ、JSで言語切替。
- 規模: 487ページのまま。アプリ本体と同じ作法で一貫。
- ⚠️ **SEOは弱い**: Googleは既定でja版を索引(クライアント描画の翻訳は索引されにくい)。**「SEO効果大」目的には不十分**。

> **推奨 = 案A**。今回コンテンツを先にやる理由がSEOなので、言語別静的URLでないと効果が出ない。案Bは「とりあえず画面上は多言語」だけでSEOの果実が取れない。

## 案Aの詳細設計

### URL / ファイル構成
```
/ability/アイスボディ.html              … ja(現状維持・既存URL不変=被リンク/索引を壊さない)
/en/ability/ice-body.html              … en
/fr/ability/ice-body.html              … fr  …(8言語)
（pokemon: /en/pokemon/<en-slug>.html、type: /en/type/<en-slug>.html も同様）
```
- ja既存URLは**変えない**(SEO資産保護)。非jaは `/<lang>/…` 配下に新設。
- slug: ability/typeは英語名→kebab(ice-body, fire)。pokemonは既存の英語slug流用。

### 各ページの翻訳点
| 要素 | ja | 非ja |
|---|---|---|
| `<html lang>` | ja | en/ko/… |
| title / meta description / og | 独自ja | 翻訳(名前+定型) |
| 見出し・パンくず・表ヘッダ等の固定文 | ja | ui-*.json |
| エンティティ名(特性/ポケ/タイプ) | DB(ja) | i18n辞書 |
| 特性の効果説明 | 独自ja | PokeAPI short_effect |
| 関連ポケ名/タイプ/分類 | DB | i18n辞書 |
| hreflang alternate | 全言語版を相互リンク | 同左 |
| 著作権/免責フッタ | ja | **ja維持**(法務=2026-06-24決定) |

### 生成器の改修方針
- `_gen_content_pages.js` を `LANGS = ['ja','en',...]` でループ。`head()`/本文ビルダーに `lang` を渡し、文字列を i18n辞書/ui-*.json から引く。
- ja は現行どおり既存パスへ。非ja は `/<lang>/…` へ出力。
- hreflang セット・sitemap(言語別エントリ)も生成。
- 既存の index.html 等トップは触らない(現行ルール踏襲)。

### 検証ゲート
- i18n監査ハーネスの PAGES に代表ページ(各言語の sample)を足し、残日本語(法務除く)0 を確認。
- 全ページ `node --check` 相当(HTML妥当性)+ サンプル実機表示。
