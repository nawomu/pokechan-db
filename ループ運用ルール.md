# ループ運用ルール（自動化でバトルシミュレーター再現を回すときの"回し方"の規律）

最終更新: 2026-06-08 JST
**これは「中身のルール」ではなく「回し方のルール」**。説明文の書き方=`review/rules.html`、声の判断例=`ヤックン耳_判断ログ.md`、憲法=`CLAUDE.md`。**それらと重複させない**。ここは「ループ/ゴールで自走させる作業をどう進め、どう引き継ぐか」だけ。

---

## ★このファイルの育て方（増やしすぎ防止の規律＝最優先）
- **1ルール = 短く1行 ＋ なぜ(1文)**。長文禁止。
- **追加する前に「既存と重複/矛盾しないか」を必ず確認** → 重複なら**統合**、矛盾なら**片方を廃止**(両立させて曖昧にしない)。
- **古い/誤ったルールは消す**(または `~~廃止~~(理由)` と明記)。死んだルールを残さない。
- **番号を振らない**(増減で崩れる)。**カテゴリで整理**。
- **増えてきたら棚卸し**(統合して減らす)。目標は"網羅"でなく"効く最小限"。
- **声・データ判断など人間専権は CLAUDE.md / rules.html / 判断ログ に置く**。ここに書くのは"プロセス"だけ。
- 新セッションは **CLAUDE.md → このファイル → 最新HANDOFF → テスト結果** を読めば、説明なしで続けられる状態を保つ。

## 自走の境界（何を自動化してよいか）
- **承認済みの計画は、実行中いちいち確認しない**(2026-06-08 阿部さん「逐一確認が渋滞」)。設計/手順に阿部さんOKが出たら、その範囲は**自走で進める**。**安全はゴールデンテスト/機械テストで担保**(壊れたら自分で気づく)。止まるのは"新たな判断点"(データの意味・声・想定外の破綻)だけ。各ステップで HANDOFF を更新し、いつ中断しても渡せる状態を保つ。
- **機械で合否が測れるもの(sim計算=テスト)だけ**自走させる。**声(説明文)・データ判断・スキーマ追加は人間チェック**＝自走に入れない(混ぜると機械が誤判定して暴走)。
- **逐次(メインループ/goal) vs ダイナミックワークフローズ の使い分け**(2026-06-08): 前段に依存する**積み上げ実装(段階開通・1ファイルを小さく直す)は逐次**(並列だと同じファイルを取り合い競合・workflowは割高)。**「広く大量に並列」できる所=ファンアウト**(多数の技の正解値生成・全技の一括監査・複数メカニクスの調査)で **workflow**。/goalのメインループから部分的にworkflowを呼ぶハイブリッドが基本。
- **Claudeは ★→✓ に上げない**。完成判定は「機械テストの緑」か「阿部さんの耳」。自分で"完了"を宣言しない。
- **/goal には必ずターン上限**を付ける(暴走・空回り防止)。

## 正解の置き方（★最重要・間違える元）
- **正解は"本物(権威ソース)"に当てて確かめる**(2026-06-08 阿部さん)。sim/データは検証対象＝正解ではない。**sim の自己出力をゴールデン(正解)にしない＝自己参照は偽の正解**。テストの期待値は**権威ソース(日本のWiki等)の既知値**から取り、出典を残す。
- **羅針盤が先・実装は後**。「何が正しいか/どこがゴールか」を `バトル再現_羅針盤.md` で定めてから動く。調べることが考えること=調べなければ正しいゴールに行かない。妄想・記憶で断言しない(Claudeはゲーム未プレイ)。

## 完了の定義（"終わった"を間違えない）
- **「穴ゼロ/カバー率/grepゼロ/全部日本語」は機械の話＝偽の頂上**。隠れ取りこぼし(音系・requires・delayed状態・on_charge_turn 等)があるので、**テスト＋人間照合**で確かめる(voiced≠complete)。
- **完了の単位は「項目が穴ゼロ＋テスト緑(＋声は耳OK)」**。穴ゼロに到達した項目は安定＝以後の再チェック不要。

## データとエンジン
- **effects はバトルシミュレーター前提**＝常に**フェーズ(優先度→攻撃前→攻撃→攻撃後→終了)**で考える。説明はそれを訳すだけ。
- **compose が"賢く"織り込み/並べ替えしたくなったら = effects の phase 設計が足りないサイン** → エンジンに特例でなく**データ(phase)を整える**。
- **SSOT(`pokechan_data.js`)編集は sim に効く** → 置換件数＋JSON妥当性を検証してから書く・**阿部さんに報告**。

## 引き継ぎ・自動継続（コンテキストが溢れても続く）
- **引き継ぐのは"会話"でなく"ファイルの状態"**。会話は圧縮/リセットされてもファイルは残る。
- **各周回(イテレーション)の終わりに HANDOFF を更新**(現在地・次の一手・直近の決定)。次の周回/新セッション/Routines実行が、それを読んで続きから入る。
- **共有doc(HANDOFF/CLAUDE.md/rules/このファイル)のコミットは阿部さんの了解**を取る(阿部さん直接指示時は除く)。

## 止まる条件（記憶で断言しない）
- **事実が要る / SSOTが矛盾 / 好みでしか決まらない / 声の判定** → **止まって阿部さんに聞く**。権威ソース(Bulbapedia/育成考察Wiki 等)で裏取りし、推測で進めない。
- **作る快感に流されない**: 「もう既存(legacy/データ)に在るのでは?」「これは機械で測れる話か?」を作業前に問う。
