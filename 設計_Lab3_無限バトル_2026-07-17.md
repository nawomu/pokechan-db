## §1 Lab-3-A: 手動⇔AI切替(いつでも・両側)

- **状態**: `labAI = { self:false, opp:false }`(真の側キー。labSwappedと無関係に保持)。リロード非保存(バトル自体消えるため)。
- **UI**: ターンインジケータ行(updateLabTurnIndicator付近)に両側分のトグルボタン `🧑手動 ⇔ 🤖AI` を常設(バトル中いつでも押せる)。押した瞬間から**次の行動収集**に反映。
- **行動収集フローの改修(核心)**:
  - 現行: showActionUI(self)→doTurn→swap→showActionUI(opp)→doTurn→unswap→executeTurn。
  - 改修: 「UIで聞くのは手動側だけ・AI側は executeTurn 直前に非スワップ状態で注入」。
    1. `labBeginTurn()`(ターン開始点=旧showActionUI呼び出し箇所): 非スワップ保証→手動側が1つ以上あれば従来フロー(self手動ならそのまま/selfがAIならswapしてoppから)。両方AIなら UI を出さず `labRunAiTurn()` へ。
    2. `doTurn` 末尾(unswap後・executeTurn前)に **AI注入**: `for (side of ['self','opp']) if (labAI[side] && 未収集) { const act=S.aiChooseAction(side); act.type==='switch' ? S.sides[side].switchChoice=act.index : S.sides[side].selectedMoveIdx=act.index; }`(real_battle 3047-3051と同型)。
    3. 手動が相手側だけの場合(self=AI, opp=手動): labBeginTurnで直接swap→oppのUI→doTurnがunswap→self分をAI注入→executeTurn。
  - **禁止**: labSwapped=true の間に aiChooseAction / S.sides[side] 直書きをしない(側逆転)。注入は必ず unswap 後。
- **両方AI=観戦モード**: `labRunAiTurn()` = 両側AI注入→executeTurn。endOfTurn後、`gameOver でなく labForcedQueue 空で両側AIのまま`なら **setTimeout(1200ms)で次ターン自動続行**。UIに `▶オート/⏸`(トグル)を出す。⏸ or どちらかを手動に戻すと停止。既定=オートON(切替直後から観察できる)。
- **AI側の死に出し**: labForcedQueue消費時、その側がAIなら party画面を出さず `S.attemptSwitch(side, 最初の生きidx, {ignoreTrapping:true, faintReplace:true})` で自動(real_battle先例=idxs[0]固定)。
- **AI側のメガ**: 扱わない(現状維持=AIはメガ予約しない)。

## §2 Lab-3-B: バトル中のポケモン追加

- **入口**: 交代画面(renderPartyScreen)の各側リスト末尾に `＋ついか` ボタン(**閲覧側にも出す**=真の側で判定して追加先を決める)。
- **フロー**: `＋ついか` → 既存 #poke-picker(openPicker流用・isPickableフィルタ)を「バトル追加モード」で開く → 種を選ぶと **エンジンbenchへ直接entry生成してpush**(編成スロットは触らない=そのバトル限り・rb_team_lab_*保存には入らない)。
- **entry生成**(BENCH_FIELDS準拠・battle_lab 2321の生成例に合わせる):
  - poke=選択種 / moves=その種のデフォルト4技(startBattleで編成スロット未設定時に使う既定ロジックを流用。無ければ習得技先頭4つ) / ability=種の特性1 / item=null / effort=0 / natureIdx=既定 / gender=既定 / currentHp=null(入場時満タン) / fainted=false / status=null / **pp=moves.map(m=>m.pp||0) ←必須**
  - 追加後 renderPartyScreen 再描画→そのまま中央エディタで技/特性/持ち物/努力値を調整可能(既存機能がエンジンentry直書きなのでそのまま効く)。
- **上限**: ハード上限なし(リストはスクロール)。ただし現実的な表示崩れ防止で1側 **12体** でボタンを隠す(定数LAB_BENCH_CAP=12)。
- **ログ**: バトルログには出さない(battle_log_i18n改修を避ける)。UI変化で分かる。

## §3 Lab-3-C: ひんし復活

3経路(すべて `currentHp=満タン; fainted=false; status=null; sleepTurns/toxicCounterリセット` の同一ヘルパー `labRevive(entry)` を通す):
1. **交代画面**: ひんしの控え(現状 disabled=2948-2949)に `♻ふっかつ` ボタン → labRevive → 再描画(以後普通に交代可能)。
2. **神の手**: HPスライダーの既存挙動(3612-3613=0%から上げるとfainted解除)を正式仕様とする(改修不要・検証だけ)。
3. **全滅時の継続(無限化の要)**: endOfTurn の dead(side) 判定(4213-4219)を改修:
   - 全滅時、従来のWIN/LOSE表示に **2ボタン**を追加: `♻ふっかつして つづける` / `おわる`(従来のgameOver)。
   - 「つづける」→ 全滅側のひんし一覧(交代画面流用 or 簡易リスト)から1体選んで labRevive → `S.attemptSwitch(side, idx, {ignoreTrapping:true, faintReplace:true})` で場に出す → gameOverフラグを立てずにバトル続行(次ターンへ)。
   - gameOverを一度も立てない経路にする(立てた後の巻き戻しはしない)。
- **注意(仕様として許容)**: おはかまいり威力は「現在ひんしの控え数」近似なので復活で変わる(ラボなのでOK・docに明記)。
