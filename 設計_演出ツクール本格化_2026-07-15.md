## §2 W3: インスペクタ本格化(本書の新規設計)

### 2-1. popnum(ダメージ数字)の大きさ・消え方を調整可能に【不満B】

**現状の構造(裏取り済み)**:
- `popText(side, text, color, size, variant, durMs)`(`fx_primitives.js:22-32`)は **size引数を既に持つ**が、ディスパッチ(`fx_primitives.js:876`)は `p.size || 16` 固定で**エディタに編集欄が無い**。
- 消え方は `@keyframes rbPop`(`real_battle.html:443-445` 他3ファイル)**1本に焼き込み**: 0%出現→18%ポップイン→100%フェード+上昇46px。`durMs` は全体を伸縮するだけで「**表示を保つ時間**」と「**消える速さ**」を独立に調整できない。

**設計**:
1. **cue.params 追加**(text/popnumキュー): 
   - `size`(px・既定16・clamp 8〜96)
   - `holdMs`(ポップイン後にその場で保つ時間・既定0)
   - `fadeMs`(消えるのにかける時間・既定=従来比率)
   - `rise`(消えながら上昇するpx・既定46)
2. **popText改修(省略時=旧挙動と完全一致が絶対条件)**: `holdMs`/`fadeMs`/`rise` の**いずれかが指定された時のみ** CSSアニメをやめて **WAAPI 3相**で組む:
   - 相1 ポップイン(約180ms固定・scale .6→1.18のバウンス=rbPopの0〜18%を再現)
   - 相2 ホールド(`holdMs`・不透明度1で静止)
   - 相3 フェード(`fadeMs`かけて opacity→0+`rise`px上昇)
   - 全パラメータ未指定(=本番の既存呼び出し・cueシート無し技の既定2000ms)は**従来のCSS `rbPop` 経路のまま1msも変えない**。
   - ★罠: rbPopは `translateX(-50%)` を含む。WAAPIキーフレームでも**毎フレームに `translateX(-50%)` を明記**しないと横位置が飛ぶ。
   - variant(crit/se)は従来CSSのまま(WAAPI化は将来。variant指定時は新パラメータを無視して従来経路でも良い=まず通常数字だけ)。
3. **ディスパッチ配線**(`_dispatchCueProd` text分岐): `p.size`(既に読んでる)に加え `p.holdMs`/`p.fadeMs`/`p.rise` を popText へ透過。
4. **エディタUI**: text/popnumキュー選択時に size / holdMs / fadeMs / rise の数値欄を追加(`renderInspector`+`wireInspectorEvents`)。プレビューは同一ディスパッチ経由なので配線だけで即時反映される。

### 2-2. burst の強度・粒をインスペクタから調整可能に

- `intensity` は**ディスパッチが既に読む**(`p.intensity || 'normal'`・`fx_primitives.js:859`)のに**UI欄が無い**だけ → glyph/burstキュー選択時に select(normal/up/crit/down)を追加。配線のみ。
- (小・任意)`particles`(粒の数の上書き・既定=intensity由来のn)。`spawnBurstParticles` の n を `p.particles` で上書きできるようにする。省略時=従来。

### 2-3. チャージ(突進)モーションのタイミング調整【不満Cの「調整」側】

**現状**: `_cueChargeMotionProd`(`fx_primitives.js:772-817`)のキーフレームは **offset 0.55(接触)/0.62(滞留)/1(帰還)・hitFrac 0.85(踏み込み距離)・easing固定**。cue.durで総尺しか変えられない。

**設計**: cue.params 追加(atk/chargeキュー)・**全て省略時=現行値**:
- `impactFrac`(既定0.55・行きにかける割合)
- `dwellFrac`(既定0.62・接触して止まっている終端。impactFrac<dwellFrac<1をclampで保証)
- `reachFrac`(既定0.85・相手へどこまで踏み込むか 0.3〜1.2)
- `easing`(プリセットselect: standard=現行 `cubic-bezier(.3,0,.2,1)` / sharp(鋭く踏み込む) / heavy(重い助走) の3種から)
- エディタ: atk/chargeキュー選択時に上記欄。**エディタ側 `cueChargeMotion` と本番 `_cueChargeMotionProd` の両方に同じ読み取りを入れる**(±ロジック同一を維持。可能なら共通化=エディタ版がプロダクション版へ委譲する形に寄せる)。
- ★クローンのscale焼き込み(`scale=r.width/offsetWidth`・transform-origin '0 0')には**触れない**(2026-07-13根治箇所)。キーフレームの transform に `scale(${scale})` を含める現行形を維持したまま offset/距離/easing だけ可変にする。

1. `node --check fx_primitives.js` → ローカルサーバ+Playwright実機。
2. **後方互換(最重要)**: cueシート無し技+新パラメータ未指定シート(フレアドライブ現行)で **popnum/burst/チャージの見た目が従来と完全一致**(before/afterスクショ比較)。
3. 新パラメータ動作: size=48の数字が大きく出る/holdMs=800で保ってから消える/fadeMs=200で素早く消える/チャージeasing 3種が見て分かる。3ページ(real_battle/online_battle/fx_editor)でJSエラー0。

## §3 実装オーケストレーション(回し方)

- **実装=Sonnetサブエージェント**に1ウェーブずつ委譲(メインrepo・worktree不要=逐次だから)。**同じファイルを触るウェーブは絶対に並列しない**。
- 各ウェーブの定型: Sonnetに手順書§を指定して委譲 → 私が**差分全文目視**([[agent-file-commit-lockout]]) → 検証ゲート(Playwright+スクショ) → `release_check` → **対象ファイルのみ`git add`** → commit → push → `?cb=`でCDN反映確認 → 阿部さんへ🙋つき報告。
- SSOT: cueの正は `reference/battle_fx_cues.json` → `node tools/_build_battle_fx_cues_js.js`(ビルド産物直編集禁止)。
- GLMを使う場合はファイル編集のみのタスク(例: 3ページのCSS同一変更)に限定し、ビルド/検証は親でやる。
