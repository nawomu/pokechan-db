# 本文ドラフト

## はじめに

はじめまして。PchamDB 管理人のぴ〜ちゃんと申します。

このたび、ポケモンチャンピオンズの**非公式ファンデータベース「PchamDB（ピーチャンディービー）」** を公開しました。
https://pchamdb.com

「いやポケモン関連のサイトなんて山ほどあるじゃない」と思われるかもしれません。実際そうなんです。それでも自分で作った理由と、**AI（Claude Code）と二人三脚で1ヶ月** という制作過程を、これから何回かに分けて書いていこうと思います。

第1回は「なぜ作ったか」と「使ったツール」の話です。

## 私はもともと、ポケモンをやったことがなかった

実は私、ポケモンと出会ったのはここ数年です。それまでプレイしたこともなく、知識もほぼゼロでした。

きっかけは、ゲームセンターの **UFO キャッチャー** でした。

子供が「ポケットモンスター サン・ムーン」のソフトを欲しがって、私は「そんなの取れるわけないよ、100円だけね」と言って軽い気持ちでやってみたところ── **100円で取れてしまった** んです。

「えっ、取れちゃったよ……」

でも当時、我が家には Nintendo DS がなかったので、ソフトはあるけどゲームができない状態。そこに知り合いが「**新しいのを買ったから、古いDS あげるよ**」とポンと譲ってくれて、これが私の最初のポケモン体験になりました。

ただ、いざ始めてみると **1本目のソフトで対戦をするのはハードルが高い** ことに気づきます。手持ちのポケモンも少なく、対戦の準備や育成方法もよく分からない。やりたくても対戦バトルができるような状況ではありませんでした。

その後「ウルトラサン・ムーン」なども購入し、少しずつポケモンが溜まっていきました。それでも育成にはとにかく時間がかかる。だんだんと、**「もっとデータでしっかり把握したい」** と思うようになり、自分でポケモンのデータを集めて比較し始めました。

そして気づいたんです。

> **データを見ているだけでも、面白い。**

ゲームを直接プレイしなくても、種族値や相性表を眺めるだけで頭の中でバトルが回る。育成計画を立てるのも、強い手持ち構成を考えるのも、データだけで完結する楽しさがあるんです。

こうして、対戦に向けたデータ整理が、私のポケモンとの主な向き合い方になっていきました。

---

## なぜ作ったのか

きっかけは単純で、**「対戦準備に欲しいツールがなかった」** からです。

ポケモンチャンピオンズが発表されて、対戦コミュニティで「あのポケモンの種族値は？」「このタイプの相性は？」「このわざ誰が覚える？」といった話題が日常的に出ます。

既存の攻略サイトはとても優秀で、情報量も網羅性も素晴らしい。でも、**対戦準備に直結する情報を一画面で横並びに見たい** という用途には少し向いていない、と私は感じていました。

- タイプ相性表
- 種族値の一覧（ソート・フィルタ可能）
- 手持ち6匹のタイプ補完チェック
- わざを威力・命中・効果で検索

これらを **一つのサイトで、なるべくクリック数少なく** 確認したかった。

「ないなら、作るしかないか」── そう思い立ったのが2026年4月のことです。

---

## 実は、すでに Google スプレッドシートで自作していた

正直に言うと、PchamDB のような対戦用ツールは、私の中で **以前から自分用に作っていた** ものでした。

- 種族値・タイプの一覧と並び替え
- 手持ち6匹のタイプ補完を確認するためのチェックリスト
- 「このポケモンの弱点はどこか？」を瞬時に見るための相性マトリクス
- 持ち物・技構成の検討メモ

全部、**Google スプレッドシート上の手作業** で作っていました。条件付き書式とフィルタ機能をフル活用して、それなりに使えてはいました。

[ここに「手持ち 技構成の検討表」のスクショ]

[ここに「全網羅の対戦用ポケモン厳選」のスクショ]

でも、スプレッドシートには **限界** があったんです。

- 行数が増えると重い、操作がもたつく
- 動的なソート・フィルタの組み合わせがしんどい
- 他の対戦勢に共有しても、編集権限まわりがややこしい
- スマホで見るとレイアウトが崩れる
- 機能を足すたびに数式が複雑化して、自分でもメンテ不能になりがち

「これって本当はウェブサイトで作るべきものだよね」とずっと思っていました。**頭の中に設計はあるのに、それを実装できない** 状態が長かったんです。

そんな時に登場した **Claude Code** で「これ、ウェブ化までイケるんじゃない？」と確信し、今回の制作に踏み切ったわけです。

---

## でも、自分はそんなにプログラミング得意じゃない

正直に言います。

私はバリバリのエンジニアではありません。プログラミングは少しかじった程度。HTMLとCSSは触れますが、JavaScriptで動的なテーブルを作って、データベースみたいに検索・ソートできるサイトを **1人でゼロから作るのはしんどい** です。

ちなみに正確に言うと、**大学では国立の情報工学科** にいたので、C や Pascal の授業を受けたことはあります。ただ、その後の仕事ではコードを書く機会がほぼなく、構文の細かい部分はもう忘れています。

それでも、「変数」「ループ」「条件分岐」みたいな **プログラミングの基礎的な考え方** だけは身体の奥にうっすら残っていて、Claude Code とのやり取りではこれが地味に役立ちました。

AI が書いてくれたコードを「ぼんやり読める」のと「全く分からない」のとでは、修正依頼の精度がだいぶ変わってきます。**完全な未経験ではないけれど、実装は AI まかせ** ── というのが私の立ち位置です。

## AI 遊びから、Claude にたどり着いた

実は私、もう1年以上前から **AI で遊ぶのが趣味** でした。

Gemini や ChatGPT で文章を書いたり情報を整理したり、**Sora** にハマって動画生成を毎日のように回したり。SF みたいなものが個人の手元で作れる感覚に感動して、暇さえあれば触っていました。

ところが、その **Sora の動画生成サービスが2026年3月で終了** してしまいました。「お気に入りのツールが消えた」と感じて、代わりになる何かを探していた頃に、Claude を本格的に試してみたんです。

そこで気づきました。

> **これ、プログラムやウェブサイトまで作れるじゃないか。**

ぼんやり「こんなの作りたい」と話しているうちに、AI が実際にコードを書いてファイルを保存してくれる。動画生成と地続きの感覚なのに、出てくるのは動くウェブサイト。

「じゃあ、何を作ろう？」

その時に思い浮かんだのが、**昔から自分でデータ整理していたポケモンの話** でした。ちょうど **「ポケモンチャンピオンズ」** が公開されたタイミングだったのも追い風で、「このタイトル用にデータを集めて形にしよう」と決めたわけです。

---

## Claude Code とは

Anthropic 社が提供する、**ターミナル上で動くAIコーディングアシスタント** です。普通のチャットAIと違うのは、

- ファイルを直接読み書きできる
- コマンドを実行できる
- Git操作までできる

つまり、**「こんな機能を作って」と伝えれば、AIがコードを書いてファイルに保存してくれる**。私は内容を確認しながら、おかしい部分を指摘するだけで進められます。

「AIに丸投げ」とは違います。私の役割は **設計判断・データ検証・最終確認** で、AIの役割は **実装作業・ドキュメント参照・コード提案** という分担です。

---

## スクショを投げるだけで解決する、これが地味にすごい

Claude を使ってみて **「これは革命的だな」** と一番感じたのは、**スクリーンショットをそのまま渡せる** ことでした。

例えば、こういう場面で本当に助かりました:

1. **海外のサイトを確認するとき** ── 英語のドキュメントや管理画面を、画面ごとスクショして「これ何が書いてあって、次は何を押せばいい？」と聞ける
2. **AdSense の申請手続きを進めるとき** ── 不慣れな画面遷移を、スクショ + 質問だけで案内してもらえる
3. **エラーメッセージや不具合で手詰まりになったとき** ── 真っ赤なエラー文章をスクショで送るだけで、原因と対処法が返ってくる

自分で調べると、Google 検索 → 似た事例を探す → 翻訳 → 試す → 違う → 別のキーワードで再検索…という長い旅が必要です。特に英語のエラー文だと、**そもそも何が問題なのかが分からない** こともしばしば。

それが、**スクショを撮って投げるだけ**。これが想像以上に大きい差でした。「分からない場面」が「数秒で解決する場面」に変わったことで、ストレスなく前に進めるんです。

## 1ヶ月で何を作ったか

ざっくりと、こんな機能を実装しました:

### 🔍 ポケモンDB
全ポケモンの種族値・タイプ・特性・覚えるわざを一覧で表示。タイプ相性表もまとめて見られる、表計算ライクなUI。

### 🛡 手持ちわざチェッカー
手持ち6匹を入れると、タイプ補完・攻撃範囲が瞬時にチェックできるツール。「うちのパーティに足りない攻撃タイプは？」が一目で分かる。

### ⚡ わざリスト
約500種類のわざを威力・命中・優先度・効果で検索・フィルタ。「混乱させるわざ誰が覚える？」みたいな逆引きも可能。

これらを **HTML + CSS + 素のJavaScript** で実装。フレームワークは使っていません。理由は単純で、**シンプルな方が壊れにくく、表示も速い** からです。

---

## 大事にしたこと

ひとつだけ、最初から決めていたルールがあります。

> **データの正確性は、AIに丸投げしない**

ポケモンの種族値・わざデータは、**公式情報源と複数の攻略サイトを突き合わせて、人間が検証してからDBに入れる** という方針を貫きました。AIは情報を「もっともらしく」生成してしまうことがあります（いわゆるハルシネーション）。ファンサイトとはいえ、データが間違っていたら使ってもらえません。

なので、データ部分は地道に手作業、コーディングは Claude Code と二人三脚 ── という分担で進めました。

## 公開してから感じたこと

サイトを公開して、まだ間もないですが、自分で対戦準備に使うたびに「便利だな」と思います。当然です。自分が欲しかったツールを作ったので。

でも、もし誰か他の対戦勢が「これ便利」と思ってくれたら、それが一番嬉しい。

そして、もう1つ強く感じていることがあります。

**自分でプログラミングを一から学んで作っていたら、ここまで── 公開してアフィリエイトに繋げて、SEO まで整える── 絶対にたどり着けなかった** ということ。

AI に頼むと、仕様を日本語で伝えるだけで実装が進み、詰まったら「なんで動かない？」とその場で相談できる。ドメイン取得・公開設定・収益化まで、一通り並走してくれる。

その全部を、**楽しみながら、1ヶ月という驚異的なスピード** で形にできました。副産物として、**Claude Code を実地で使いこなすスキル** も少しずつ身についていく実感があります。

これからは AI の時代だな、と肌で感じる1ヶ月でした。
