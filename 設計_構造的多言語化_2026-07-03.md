# 構造的多言語化リファクタ 設計書 (2026-07-03)

## 概要
「絶対多言語になる前提」の構造に統一。挙動を変えずに構造を一本化。

## 4本柱

### 柱1: エンティティ辞書ビルダー
- ファイル: `tools/build_i18n_entities.js`
- ソース: `reference/master_{pokemon,moves,abilities,items}.json`
- 出力: `i18n/{lang}.json` (en/fr/de/es/it/ko/zh-Hans/zh-Hant)
- 実行: `node tools/build_i18n_entities.js [--dry-run] [--lang=en,fr,...] [--overwrite]`
- 冪等: 2回実行で差分ゼロ保証
- 欠落レポート: 実行時に言語別jaフォールバック件数を出力

### 柱2: 共通アクセサ
- ファイル: `i18n/runtime.js`
- API:
  - 既存: `I18N.pokemon(ja)` / `I18N.move(key, fb)` / `I18N.ability(ja)` / `I18N.item(ja)` / `I18N.nature(ja)` / `I18N.type(ja, fmt)` / `I18N.moveDesc()` / `I18N.abilityDesc()` / `I18N.itemDesc()`
  - 追加(2026-07-03): `I18N.name(kind, id)` / `I18N.desc(kind, id)` — エンティティ種別を引数で切り替える統合アクセサ
  - kind一覧: `'pokemon' | 'move' | 'ability' | 'item' | 'nature' | 'type'`
- 配線ルール(新規ページ必須): データ名はHTML直書き禁止 → `I18N.name(kind, id)` 経由

### 柱3: タグ語彙テーブル駆動チップ
- ファイル: `i18n/move_tags_i18n.json`
- waza_picker.js の `setupNewTagFilter()` が `MOVE_TAG_TR` を参照して翻訳
- 新規タグ追加手順: `move_tags_i18n.json` に9言語セットで追加 → チップは自動追加
- 参照方式: JA絵文字タグ文字列をキーとして各言語訳を参照

### 柱4: 監査ゲート
- ファイル: `tools/i18n_audit_playwright.js`
- 対象: 25ページ(19→拡張)
- 言語: デフォルト全9言語
- 許可リスト: `ALLOWLIST`(ポケモンチャンネル/ヤックン/ポケモン等ブランド名)
- `--strict` モード: 許可リスト外残日本語0を期待、1件でもあればexit 1
- push前チェック: `bash tools/release_check.sh`

## 新規ページ追加チェックリスト
1. HTMLに `<script defer src="i18n/runtime.js">` を追加
2. 静的UI文言は `data-i18n="key"` 属性を使う(ハードコードのJAリテラル禁止)
3. データ名(ポケモン名/技名/特性名等)は `I18N.name(kind, id)` 経由でレンダリング
4. プレースホルダー付き文は `data-i18n-attr="placeholder:key"` を使う
5. 語順が変わる文は `I18N.t('key', 'ja文{0}')` でプレースホルダーを使う
6. `i18n/ui-*.json` の9言語に新規キーを追加(JAとEN/FR/DE/ES/IT/KO/ZH-HANS/ZH-HANTのセット)
7. `bash tools/release_check.sh` でゲート確認

## 辞書ファイル一覧
| ファイル | 用途 | 生成方法 |
|---|---|---|
| `i18n/{lang}.json` | エンティティ辞書(ポケモン名/技名/特性名/道具名/性格名) | `node tools/build_i18n_entities.js` |
| `i18n/ui-{lang}.json` | UI固定文言(ボタン/ラベル/メッセージ) | 手動管理(9言語セット) |
| `i18n/types-master.json` | タイプ名(full/short3形式) | 手動管理 |
| `i18n/move_tags_i18n.json` | 技タグ語彙(絵文字付きJAキー→各言語) | 手動管理 |
| `i18n/move_desc_own.json` | 独自翻訳技説明文 | 別管理(翻訳WF) |

## 禁止事項
- HTMLにJA文字列を直書き(data-i18n/I18N経由にする)
- エンティティ辞書を手動で直接編集(build_i18n_entities.js 経由に統一)
- verbatim一致翻訳の流用(北極星ルール・CLAUDE.md参照)
